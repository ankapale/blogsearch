max-height: 528px;"><div class="aspectRatioPlaceholder-fill" style="padding-bottom: 75.4%;"></div><img class="graf-image" data-image-id="1*8SAi0i2oZbxxMVc-EOfovA.png" data-width="953" data-height="719" data-action="zoom" data-action-value="1*8SAi0i2oZbxxMVc-EOfovA.png" src="https://cdn-images-1.medium.com/max/1600/1*8SAi0i2oZbxxMVc-EOfovA.png"></div></figure><h3 name="fed1" id="fed1" class="graf graf--h3 graf-after--figure">Where to start?</h3><p name="6efa" id="6efa" class="graf graf--p graf-after--h3">There’s a lot to this app, let’s begin with what it does. <br>(Check the bottom of the article for a TLDR)</p><p name="823d" id="823d" class="graf graf--p graf-after--p">Discord Bot Maker is still somewhat limited, and in earlier development, but it can do a lot of basic text commands, and event commands; it’s just limited in scope of how much exactly it can do.<br>Some of the features include:</p><ul class="postList"><li name="fc27" id="fc27" class="graf graf--li graf-after--p">Running the bot project via App</li><li name="2b2f" id="2b2f" class="graf graf--li graf-after--li">Auto-restart ability when you save</li><li name="4b42" id="4b42" class="graf graf--li graf-after--li">Basic command creation</li><li name="d42f" id="d42f" class="graf graf--li graf-after--li">Basic event creation</li><li name="31de" id="31de" class="graf graf--li graf-after--li">Ability to be more advanced with Variables</li><li name="ac8e" id="ac8e" class="graf graf--li graf-after--li">Steam achievements (<em class="markup--em markup--li-em">We need our achievements</em>)</li></ul><p name="c5c8" id="c5c8" class="graf graf--p graf-after--li">The biggest issue from it comes not from the functionality, which is growing every day, but in the usability. <br>The app is catered to normal users but currently without documentation or better tutorials the app can be very confusing to a user using it.</p><figure name="4edf" id="4edf" class="graf graf--figure graf--layoutOutsetLeft graf-after--p"><div class="aspectRatioPlaceholder is-locked" style="max-width: 433px; max-height: 233px;"><div class="aspectRatioPlaceholder-fill" style="padding-bottom: 53.800000000000004%;"></div><img class="graf-image" data-image-id="1*NPXoO0GuHR4NQ5EgeQg4FA.png" data-width="433" data-height="233" src="https://cdn-images-1.medium.com/max/1200/1*NPXoO0GuHR4NQ5EgeQg4FA.png"></div></figure><p name="b5e2" id="b5e2" class="graf graf--p graf-after--figure">For all basic functionality it does well, and the templates you can pick from all work well; just needs refinement.</p><h3 name="ce4c" id="ce4c" class="graf graf--h3 graf-after--p">What about developers?</h3><p name="366d" id="366d" class="graf graf--p graf-after--h3">So… an app to build bots. How does this impact developers?</p><p name="d621" id="d621" class="graf graf--p graf-after--p">Well, currently there’s not much of a threat to the <a href="https://www.carbonitex.net/discord/bots" data-href="https://www.carbonitex.net/discord/bots" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">bigger public bots</a>. It’d take a decent bit of work to make most of their functionality with this, if you could even achieve that fully. <br>Those bots have an edge over DBM, but it’s a great alternative for people trying to express their own creative freedom with their bots; Unique responses, Icons, and the bots name. That’s something no developer can overcome without running a bot on a million different tokens/accounts.</p><p name="5698" id="5698" class="graf graf--p graf-after--p">Will DBM be something bot devs need to worry about in the future? Probably. It’s like having the bot <a href="https://github.com/Cog-Creators/Red-DiscordBot" data-href="https://github.com/Cog-Creators/Red-DiscordBot" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">Red</a>, but with no need to learn how to set it up and ability to build it yourself as you go. <br><strong class="markup--strong markup--p-strong"><em class="markup--em markup--p-em">Feel accomplished</em></strong><em class="markup--em markup--p-em"> + </em><strong class="markup--strong markup--p-strong"><em class="markup--em markup--p-em">Get customization</em></strong><em class="markup--em markup--p-em"> + </em><strong class="markup--strong markup--p-strong"><em class="markup--em markup--p-em">One time fee</em></strong><em class="markup--em markup--p-em"> = </em><strong class="markup--strong markup--p-strong"><em class="markup--em markup--p-em">Profit</em></strong><br>It’s hard to not want to go for a 10$ one time payment than pay for <a href="https://dynobot.net/upgrade" data-href="https://dynobot.net/upgrade" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">some premium bot</a> to get your 100% uptime or whatever it is you want.</p><p name="e92f" id="e92f" class="graf graf--p graf-after--p">Discord Bot Maker is not a tool to replace coding, it’s a tool to give people who can’t/won’t code a chance to do their own thing without being limited to the options given to them by public bot devs. Did I mention it’s built in <a href="https://discord.js.org/" data-href="https://discord.js.org/" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">Discord.js</a>?</p><h3 name="7921" id="7921" class="graf graf--h3 graf-after--p">Where’s the app going?</h3><p name="ce95" id="ce95" class="graf graf--p graf-after--h3">The app is going somewhere, it’s a <a href="https://en.wikipedia.org/wiki/Content_management_system" data-href="https://en.wikipedia.org/wiki/Content_management_system" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">CMS</a> for bots.</p><p name="d614" id="d614" class="graf graf--p graf-after--p">Have you seen people pay for plans on <a href="http://webs.com" data-href="http://webs.com" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">Webs</a>, <a href="http://wordpress.com" data-href="http://wordpress.com" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">Wordpress</a>, <a href="http://wix.com" data-href="http://wix.com" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">Wix</a>, etc? Well, this is the same concept except a one time fee. “Pay this and make a bot, no coding required!” that’s just asking for people to throw their money at it.</p><p name="d145" id="d145" class="graf graf--p graf-after--p">I had a meeting with the dev and his intentions are good; it’s something he does for fun, not for the pricetag.</p><p name="a5e8" id="a5e8" class="graf graf--p graf-after--p">If you had the chance to give people more creative freedom, would you? Not if you’re a big bot dev! And that’s ok, bots like <a href="http://dynobot.net" data-href="http://dynobot.net" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">Dyno</a> for moderator , <a href="http://tatsumaki.xyz" data-href="http://tatsumaki.xyz" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">Tatsumaki</a> for utility, <a href="https://kawaiibot.xyz/" data-href="https://kawaiibot.xyz/" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">KawaiiBot</a> for misc, <a href="http://guilded.gg/discord_bots" data-href="http://guilded.gg/discord_bots" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">Guilded</a> for team/event organizing, and others will always be bots I love to see on servers and many of them can’t be replaced with something like this; they’re too complex and unique for that.</p><h3 name="65e7" id="65e7" class="graf graf--h3 graf-after--p">So what do I think?</h3><p name="084d" id="084d" class="graf graf--p graf-after--h3">Guess I should probably explain what I think!</p><p name="ce44" id="ce44" class="graf graf--p graf-after--p">The Discord Bot Maker app is not something I’d currently pour hundreds of dollars into but, as the next few weeks more should be added to it, it’ll be worth the 10$ in no time.</p><p name="8c2c" id="8c2c" class="graf graf--p graf-after--p">It’s already very functional and can achieve more than a basic discord.js tutorial bot, this could speed up the bot making process for people who can’t (<em class="markup--em markup--p-em">or don’t have the time to</em>) code.</p><figure name="7890" id="7890" class="graf graf--figure graf-after--p"><div class="aspectRatioPlaceholder is-locked" style="max-width: 700px; max-height: 381px;"><div class="aspectRatioPlaceholder-fill" style="padding-bottom: 54.50000000000001%;"></div><img class="graf-image" data-image-id="1*y6STgYkYebVhedZLBfR_sw.png" data-width="1358" data-height="740" data-is-featured="true" data-action="zoom" data-action-value="1*y6STgYkYebVhedZLBfR_sw.png" src="https://cdn-images-1.medium.com/max/1600/1*y6STgYkYebVhedZLBfR_sw.png"></div></figure><p name="6f67" id="6f67" class="graf graf--p graf-after--figure">So would I recommend it? Yes, I think developers could find a lot of use studying it. I <strong class="markup--strong markup--p-strong">don’t</strong> think every user should start clamoring to buy it until more features come out, docs become a thing, and the tutorials improve.</p><p name="7878" id="7878" class="graf graf--p graf-after--p"><strong class="markup--strong markup--p-strong">Tldr;<em class="markup--em markup--p-em"> 7.3/10<br>Decent app, but it needs a lot of work;
