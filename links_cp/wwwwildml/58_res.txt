<div class="entry-content">
<p><strong><a href="https://github.com/dennybritz/nn-theano">Get the code: The full code is available as an Jupyter/iPython Notebook on Github! </a></strong></p>
<p>In a <a href="http://www.wildml.com/2015/09/implementing-a-neural-network-from-scratch/">previous blog post</a> we build a simple Neural Network from scratch. Let&#8217;s build on top of this and speed up our code using the <a href="http://deeplearning.net/software/theano/">Theano</a> library. With Theano we can make our code not only faster, but also more concise!</p>
<h2>What is Theano?</h2>
<p>Theano describes itself as a Python library that lets you to define, optimize, and evaluate mathematical expressions, especially ones with multi-dimensional arrays. The way I understand Theano is that it allows me to define <strong>graphs of computations</strong>. Under the hood Theano optimizes these computations in a <a href="http://deeplearning.net/software/theano/introduction.html#introduction">variety of ways</a>, including avoiding redundant calculations, generating optimized C code, and (optionally) using the GPU. Theano also has the capability to automatically <a href="http://deeplearning.net/software/theano/tutorial/gradients.html">differentiate</a> mathematical expressions. By modeling computations as graphs it can calculate complex gradients using the chain rule. This means we no longer need to compute the gradients ourselves!</p>
<p><span id="more-74"></span></p>
<p>Because Neural Networks are easily expressed as graphs of computations, Theano is a great fit. It&#8217;s probably the main use case and Theano includes several <a href="http://deeplearning.net/software/theano/library/tensor/nnet/index.html#module-nnet">convenience functions for neural networks</a>.</p>
<h2>The Setup</h2>
<p>The setup is identical to that in <a href="http://www.wildml.com/2015/09/implementing-a-neural-network-from-scratch/">Implementing a Neural Network from Scratch</a>, which I recommend you read (or at least skim) first. I&#8217;ll just quickly recap: We have two classes (red and blue) and want to train a Neural Network classifier that separates the two. We will train a 3-layer Neural Network, with input layer size 2, output layer size 2, and hidden layer size 3. We will use batch gradient descent with a fixed learning rate to train our network.</p>
<p><a href="http://www.wildml.com/wp-content/uploads/2015/09/nn-from-scratch-dataset.png"><img class="alignnone size-full wp-image-10" src="http://www.wildml.com/wp-content/uploads/2015/09/nn-from-scratch-dataset.png" alt="A moon-shaped dataset with two classes." width="606" height="479" srcset="http://www.wildml.com/wp-content/uploads/2015/09/nn-from-scratch-dataset.png 606w, http://www.wildml.com/wp-content/uploads/2015/09/nn-from-scratch-dataset-300x237.png 300w" sizes="(max-width: 606px) 100vw, 606px" /></a></p>

<h2>Running on a GPU</h2>
<p>To speed things up further Theano can <a href="http://deeplearning.net/software/theano/tutorial/using_gpu.html">make use of your GPU</a>. Theano relies on Nvidia&#8217;s CUDA platform, so yo need a <a href="http://docs.nvidia.com/cuda/cuda-getting-started-guide-for-mac-os-x/#cuda-enabled-gpu">CUDA-enabled graphics card</a>. If you don&#8217;t have one (for example if you&#8217;re on a Macbook with an integrated GPU like me) you could spin up a <a href="https://aws.amazon.com/ec2/instance-types/#gpu">GPU-optimized Amazon EC2 instance</a> and try things there. To use the GPU with Theano you must have the CUDA toolkit installed and <a href="http://deeplearning.net/software/theano/install.html#gpu-linux">set a few environment variables</a>. The <a href="https://github.com/dennybritz/nn-theano">repository readme</a> contains instructions on how to setup CUDA on an EC2 isntance.</p>
<p>While any Theano code will run just fine with the GPU enabled you probably want to spend time optimizing your code get optimal performance. Running code that isn&#8217;t optimized for a GPU may actually be slower on a GPU than a CPU due to excessive data transfers.</p>
<p>Here are some things you must consider when writing Theano code for a GPU:</p>
<ul>
<li>The default floating point data type is <code>float64</code>, but in order to use the GPU you must use <code>float32</code>. There are several ways to do this in Theano. You can either use the <a href="http://deeplearning.net/software/theano/library/tensor/basic.html#all-fully-typed-constructors">fulled typed tensor constructors</a> like <code>T.fvector</code> or set <code>theano.config.floatX</code> to <code>float32</code> and use the <a href="http://deeplearning.net/software/theano/library/tensor/basic.html#constructors-with-optional-dtype">simple tensor constructors</a> like <code>T.vector</code>.</li>
<li>Shared variables must be initialized with values of type <code>float32</code> in order to be stored on the GPU. In other words, when creating a shared variable from a numpy array you must initialize the array with the <code>dtype=float32</code> argument, or cast it using the <code>astype</code> function. See the <a href="http://docs.scipy.org/doc/numpy/user/basics.types.html">numpy data type documentation</a> for more details.</li>
<li>Be very careful with data transfers to and from the GPU. Ideally you want all your data on the GPU. Ways to achieve this include initializing commonly used data as shared variables with a <code>float32</code> data type, and to <a href="http://deeplearning.net/software/theano/tutorial/using_gpu.html#returning-a-handle-to-device-allocated-data">avoid copying return values</a> back to the host using the <code>gpu_from_host</code> function. Theano includes <a href="http://deeplearning.net/software/theano/tutorial/profiling.html">profiling capabilities</a> that allow you to debug how much time you are spending on data transfers.</li>
</ul>

<h3>Running and Evaluation</h3>
<p>To use a GPU you must run the code with the <code>THEANO_FLAGS=device=gpu,floatX=float32</code> environment variable set. I tested the GPU-optimized code on a <code>g2.2xlarge</code> AWS EC2 instance. Running one <code>gradient_step()</code> on the CPU took around <strong>250ms</strong>. With the GPU enabled it merely took <strong>7.5ms</strong>. That&#8217;s a 40x speedup, and if our dataset or parameter space were larger we would probably see an even more significant speedup.</p>
