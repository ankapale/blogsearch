<div id="ban3_av"></div><h2>Introduction</h2>
<p>Every once in a while, a python library is developed that has the potential of changing the landscape in the field of deep learning. PyTorch is one such library.</p>
<p>In the last few weeks, I have been dabbling a bit in PyTorch. I have been blown away by how easy it is to grasp. Among the various deep learning libraries I have used till date &#8211; PyTorch has been the most flexible and effortless of them all.</p>
<p><img class="aligncenter wp-image-41792 size-medium" src="https://s3-ap-south-1.amazonaws.com/av-blog-media/wp-content/uploads/2018/02/pytorch-logo-flat-300x210.png" alt="" width="300" height="210" srcset="https://s3-ap-south-1.amazonaws.com/av-blog-media/wp-content/uploads/2018/02/pytorch-logo-flat-300x210.png 300w, https://s3-ap-south-1.amazonaws.com/av-blog-media/wp-content/uploads/2018/02/pytorch-logo-flat.png 400w" sizes="(max-width: 300px) 100vw, 300px" /></p>
<p>In this article, we will explore PyTorch with a more hands-on approach, covering the basics along with a case study. We will also compare a neural network built from scratch in both numpy and PyTorch to see their similarities in implementation.</p>
<p>Let&#8217;s get on with it!</p>
<p><em>Note &#8211; This article assumes that you have a basic understanding of deep learning. If you want to get up to speed with deep learning, please go through <a href="https://www.analyticsvidhya.com/blog/2016/03/introduction-deep-learning-fundamentals-neural-networks/">this article first</a>.</em></p>
<p>&nbsp;</p>
<h2>Table of Contents</h2>
<ul>
<li>An Overview of PyTorch</li>
<li>Diving into the Technicalities</li>
<li>Building a neural network in Numpy vs. PyTorch</li>
<li>Comparison with other deep learning libraries</li>
<li>Case Study &#8211; Solving an image recognition problem with PyTorch</li>
</ul>
<p>&nbsp;</p>
<h2>An Overview of PyTorch</h2>
<p>PyTorch&#8217;s creators say that they have a philosophy &#8211; they want to be imperative. This means that we run our computation immediately. This fits right into the python programming methodology, as <strong>we don&#8217;t have to wait for the whole code to be written before getting to know if it works or not. We can easily run a part of the code and inspect it in real time.</strong> For me as a neural network debugger, this is a blessing!</p>
<p>PyTorch is a python based library built to provide flexibility as a deep learning development platform. The workflow of PyTorch is as close as you can get to python&#8217;s scientific computing library &#8211; numpy.</p>
<p>Now you might ask, why would we use PyTorch to build deep learning models? I can list down three things that might help answer that:</p>
<ul>
<li><strong>Easy to use API</strong> &#8211; It is as simple as python can be.</li>
<li><strong>Python support</strong> &#8211; As mentioned above, PyTorch smoothly integrates with the python data science stack. It is so similar to numpy that you might not even notice the difference.</li>
<li><strong>Dynamic computation graphs </strong>&#8211; Instead of predefined graphs with specific functionalities, PyTorch provides a framework for us to build computational graphs as we go, and even change them during runtime.This is valuable for situations where we don&#8217;t know how much memory is going to be required for creating a neural network.</li>
</ul>
<p>A few other advantages of using PyTorch are it&#8217;s multiGPU support, custom data loaders and simplified preprocessors.</p>
<p>Since its release in the start of January 2016, many researchers have adopted it as a go-to library because of its ease of building novel and even extremely complex graphs. Having said that, there is still some time before PyTorch is adopted by the majority of data science practitioners due to it&#8217;s new and &#8220;under construction&#8221; status.</p>
<p>&nbsp;</p>
<h2>Diving into the Technicalities</h2>
<p>Before diving into the details, let us go through the workflow of PyTorch.</p>
<p>PyTorch uses an imperative / eager paradigm. That is, each line of code required to build a graph defines a component of that graph. We can independently perform computations on these components itself, even before your graph is built completely. <strong>This is called &#8220;define-by-run&#8221; methodology.</strong></p>
<p><img class="aligncenter wp-image-41714" src="https://s3-ap-south-1.amazonaws.com/av-blog-media/wp-content/uploads/2018/02/dynamic_graph-300x169.gif" alt="" width="550" height="310" srcset="https://s3-ap-south-1.amazonaws.com/av-blog-media/wp-content/uploads/2018/02/dynamic_graph-300x169.gif 300w, https://s3-ap-south-1.amazonaws.com/av-blog-media/wp-content/uploads/2018/02/dynamic_graph-768x432.gif 768w, https://s3-ap-south-1.amazonaws.com/av-blog-media/wp-content/uploads/2018/02/dynamic_graph-850x478.gif 850w, https://s3-ap-south-1.amazonaws.com/av-blog-media/wp-content/uploads/2018/02/dynamic_graph-257x144.gif 257w" sizes="(max-width: 550px) 100vw, 550px" /></p>
<p style="text-align: center;">Source:<a class="markup--anchor markup--figure-anchor" href="http://pytorch.org/about/" target="_blank" rel="nofollow noopener" data-href="http://pytorch.org/about/">http://pytorch.org/about/</a></p>
<p>Installing PyTorch is pretty easy. You can follow the steps mentioned in the <a href="http://pytorch.org/" target="_blank" rel="noopener">official docs</a> and run the command as per your system specifications.For example, this was the command I used on the basis of the options I chose:</p>
<p><img class="aligncenter wp-image-41718" src="https://s3-ap-south-1.amazonaws.com/av-blog-media/wp-content/uploads/2018/02/3-300x144.png" alt="" width="617" height="296" srcset="https://s3-ap-south-1.amazonaws.com/av-blog-media/wp-content/uploads/2018/02/3-300x144.png 300w, https://s3-ap-south-1.amazonaws.com/av-blog-media/wp-content/uploads/2018/02/3-768x368.png 768w, https://s3-ap-south-1.amazonaws.com/av-blog-media/wp-content/uploads/2018/02/3-850x407.png 850w, https://s3-ap-south-1.amazonaws.com/av-blog-media/wp-content/uploads/2018/02/3.png 1125w" sizes="(max-width: 617px) 100vw, 617px" /></p>
<pre>conda install pytorch torchvision cuda91 -c pytorch</pre>
<p>The main elements we should get to know when starting out with PyTorch are:</p>
<ul>
<li>PyTorch Tensors</li>
<li>Mathematical Operations</li>
<li>Autograd module</li>
<li>Optim module and</li>
<li>nn module</li>
</ul>
<p>Below, we&#8217;ll take a look at each one in some detail.</p>
<p>&nbsp;</p>
<h3>PyTorch Tensors</h3>
<p>Tensors are nothing but multidimensional arrays. Tensors in PyTorch are similar to numpy&#8217;s ndarrays, with the addition being that Tensors can also be used on a GPU.PyTorch supports <a href="http://pytorch.org/docs/0.3.1/tensors.html" target="_blank" rel="noopener">various types of Tensors</a>.</p>
<p>You can define a simple one dimensional matrix as below:</p>
<pre># import pytorch
import torch

# define a tensor
torch.FloatTensor([2])</pre>
<pre> 2
[torch.FloatTensor of size 1]</pre>
<p>&nbsp;</p>
<h3>Mathematical Operations</h3>
<p>As with numpy, it is very crucial that a scientific computing library has efficient implementations of mathematical functions. PyTorch gives you a similar interface, with <strong>more than 200+ mathematical operations</strong> you can use.</p>
<p>Below is an example of a simple addition operation in PyTorch:</p>
<pre>a = torch.FloatTensor([2])
b = torch.FloatTensor([3])

a + b</pre>
<pre> 5
[torch.FloatTensor of size 1]</pre>
<p>Doesn&#8217;t this look like a quinessential python approach? We can also perform various matrix operations on the PyTorch tensors we define. For example, we&#8217;ll transpose a two dimensional matrix:</p>
<pre>matrix = torch.randn(3, 3)
matrix

0.7162 1.0152 1.1525
-0.3503 -0.9452 -1.0861
-0.1093 -0.0927 -0.0476
[torch.FloatTensor of size 3x3]

matrix.t()

0.7162 -0.3503 -0.1093
 1.0152 -0.9452 -0.0927
 1.1525 -1.0861 -0.0476
[torch.FloatTensor of size 3x3]</pre>
<p>&nbsp;</p>
<h3>Autograd module</h3>
<p>PyTorch uses a technique called<a href="https://en.wikipedia.org/wiki/Automatic_differentiation" target="_blank" rel="noopener">automatic differentiation</a>. That is, we have a recorder that records what operations we have performed, and then it replays it backward to compute our gradients. This technique is especially powerful when building neural networks, as we save time on one epoch by calculating differentiation of the parameters at the forward pass itself.</p>
<p><img class="aligncenter size-full wp-image-41720" src="https://s3-ap-south-1.amazonaws.com/av-blog-media/wp-content/uploads/2018/02/4.png" alt="" width="184" height="125" /></p>

<h3>nn module</h3>
<p>PyTorch autograd makes it easy to define computational graphs and take gradients, but raw autograd can be a bit too low-level for defining complex neural networks. This is where the nn module can help.</p>
<p>The nn package defines a set of modules, which we can think of as a neural network layer that produces output from input and may have some trainable weights.</p>
<p><strong>You can consider a nn module as the <a href="https://keras.io/" target="_blank" rel="noopener">keras</a> of PyTorch!</strong></p>
<pre>import torch

# define model

 torch.nn.Linear(input_num_units, hidden_num_units),
 torch.nn.ReLU(),
 torch.nn.Linear(hidden_num_units, output_num_units),
)
loss_fn = torch.nn.CrossEntropyLoss()</pre>

<h2>Building a neural network in Numpy vs. PyTorch</h2>
<p>I have mentioned previously that PyTorch and Numpy are remarkably similar. Let&#8217;s look at why. In this section, we&#8217;ll see an implementation of a simple neural network to solve a binary classification problem (you can <a href="https://www.analyticsvidhya.com/blog/2017/05/neural-network-from-scratch-in-python-and-r/" target="_blank" rel="noopener">go through this article</a> for it&#8217;s in-depth explanation).</p>
<pre>## Neural network in numpy

import numpy as np

#Input array
X=np.array([[1,0,1,0],[1,0,1,1],[0,1,0,1]])

#Output
y=np.array([[1],[1],[0]])

#Sigmoid Function
def sigmoid (x):
 return 1/(1 + np.exp(-x))

#Derivative of Sigmoid Function
def derivatives_sigmoid(x):
 return x * (1 - x)

#Variable initialization
epoch=5000 #Setting training iterations
lr=0.1 #Setting learning rate
inputlayer_neurons = X.shape[1] #number of features in data set
hiddenlayer_neurons = 3 #number of hidden layers neurons
output_neurons = 1 #number of neurons at output layer

#weight and bias initialization
wh=np.random.uniform(size=(inputlayer_neurons,hiddenlayer_neurons))
bh=np.random.uniform(size=(1,hiddenlayer_neurons))
wout=np.random.uniform(size=(hiddenlayer_neurons,output_neurons))
bout=np.random.uniform(size=(1,output_neurons))

for i in range(epoch):
  #Forward Propogation
  hidden_layer_input1=np.dot(X,wh)
  hidden_layer_input=hidden_layer_input1 + bh
  hiddenlayer_activations = sigmoid(hidden_layer_input)
  output_layer_input1=np.dot(hiddenlayer_activations,wout)
  output_layer_input= output_layer_input1+ bout
  output = sigmoid(output_layer_input)

  #Backpropagation
  E = y-output
  slope_output_layer = derivatives_sigmoid(output)
  slope_hidden_layer = derivatives_sigmoid(hiddenlayer_activations)
  d_output = E * slope_output_layer
  Error_at_hidden_layer = d_output.dot(wout.T)
  d_hiddenlayer = Error_at_hidden_layer * slope_hidden_layer
  wout += hiddenlayer_activations.T.dot(d_output) *lr
  bout += np.sum(d_output, axis=0,keepdims=True) *lr
  wh += X.T.dot(d_hiddenlayer) *lr
  bh += np.sum(d_hiddenlayer, axis=0,keepdims=True) *lr

print('actual :\n', y, '\n')
print('predicted :\n', output)</pre>
<p>Now, try to spot the difference in a super simple implementation of the same in PyTorch (the differences are mentioned in bold in the below code).</p>
<pre>## neural network in pytorch
<strong>import torch</strong>

#Input array
X = <strong>torch.Tensor</strong>([[1,0,1,0],[1,0,1,1],[0,1,0,1]])

#Output
y = <strong>torch.Tensor</strong>([[1],[1],[0]])

#Sigmoid Function
def sigmoid (x):
  return 1/(1 + <strong>torch.exp</strong>(-x))

#Derivative of Sigmoid Function
def derivatives_sigmoid(x):
  return x * (1 - x)

#Variable initialization
epoch=5000 #Setting training iterations
lr=0.1 #Setting learning rate
inputlayer_neurons = X.shape[1] #number of features in data set
hiddenlayer_neurons = 3 #number of hidden layers neurons
output_neurons = 1 #number of neurons at output layer

#weight and bias initialization
wh=<strong>torch.randn</strong>(inputlayer_neurons, hiddenlayer_neurons)<strong>.type(torch.FloatTensor)</strong>
bh=<strong>torch.randn</strong>(1, hiddenlayer_neurons)<strong>.type(torch.FloatTensor)</strong>
wout=<strong>torch.randn</strong>(hiddenlayer_neurons, output_neurons)
bout=<strong>torch.randn</strong>(1, output_neurons)

for i in range(epoch):

  #Forward Propogation
  hidden_layer_input1 = <strong>torch.mm</strong>(X, wh)
  hidden_layer_input = hidden_layer_input1 + bh
  hidden_layer_activations = sigmoid(hidden_layer_input)
 
  output_layer_input1 = <strong>torch.mm</strong>(hidden_layer_activations, wout)
  output_layer_input = output_layer_input1 + bout
  output = sigmoid(output_layer_input1)

  #Backpropagation
  E = y-output
  slope_output_layer = derivatives_sigmoid(output)
  slope_hidden_layer = derivatives_sigmoid(hidden_layer_activations)
  d_output = E * slope_output_layer
  Error_at_hidden_layer = <strong>torch.mm</strong>(d_output, wout.t())
  d_hiddenlayer = Error_at_hidden_layer * slope_hidden_layer
  wout += <strong>torch.mm</strong>(hidden_layer_activations.t(), d_output) *lr
  bout += d_output.sum() *lr
  wh += <strong>torch.mm</strong>(X.t(), d_hiddenlayer) *lr
  bh += d_output.sum() *lr
 
print('actual :\n', y, '\n')
print('predicted :\n', output)</pre>

<h2 id="4f11" class="graf graf--h3 graf-after--li">Comparison with other deep learning libraries</h2>

<p>The APIs for data loading are well designed in PyTorch. The interfaces are specified in a dataset, a sampler, and a data loader.</p>
<p>On comparing the tools for data loading in TensorFlow (readers, queues, etc.), I found PyTorch&#8217;s data loading modules pretty easy to use. Also, PyTorch is seamless when we try to build a neural network, so we don&#8217;t have to rely on third party high-level libraries like keras.</p>
<p>On the other hand, I would not yet recommend using PyTorch for deployment. PyTorch is yet to evolve. <p><em>Our problem is an image recognition problem, to identify digits from a given 28 x 28 image. We have a subset of images for training and the rest for testing our model.</em></p>

<h3>STEP 2: Model Building</h3>
<p>a) Now comes the main part! Let us define our neural network architecture. We define a neural network with 3 layers input, hidden and output. The number of neurons in input and output are fixed, as the input is our 28 x 28 image and the output is a 10 x 1 vector representing the class. We take 50 neurons in the hidden layer. Here, we use <a href="https://arxiv.org/abs/1412.6980" target="_blank" rel="nofollow noopener">Adam</a> as our optimization algorithms, which is an efficient variant of Gradient Descent algorithm.</p>
<pre>import torch
from torch.autograd import Variable</pre>
<pre># number of neurons in each layer
input_num_units = 28*28
hidden_num_units = 500
output_num_units = 10

# set remaining variables
epochs = 5

learning_rate = 0.001</pre>
<p>b) Its time to train our model</p>
<pre># define model
model = torch.nn.Sequential(
  torch.nn.Linear(input_num_units, hidden_num_units),
  torch.nn.ReLU(),
  torch.nn.Linear(hidden_num_units, output_num_units),
)
loss_fn = torch.nn.CrossEntropyLoss()

# define optimization algorithm
optimizer = torch.optim.Adam(model.parameters(), lr=learning_rate)</pre>
<pre>## helper functions
# preprocess a batch of dataset
def preproc(unclean_batch_x):
  """Convert values to range 0-1"""
  temp_batch = unclean_batch_x / unclean_batch_x.max()
 
  return temp_batch

# create a batch
def batch_creator(batch_size):
  dataset_name = 'train'
  dataset_length = train_x.shape[0]
  
  batch_mask = rng.choice(dataset_length, batch_size)
  
  batch_x = eval(dataset_name + '_x')[batch_mask]
  batch_x = preproc(batch_x)
  
  if dataset_name == 'train':
    batch_y = eval(dataset_name).ix[batch_mask, 'label'].values
  
  return batch_x, batch_y</pre>
<pre># train network
total_batch = int(train.shape[0]/batch_size)

for epoch in range(epochs):
  avg_cost = 0
  for i in range(total_batch):
    # create batch
    batch_x, batch_y = batch_creator(batch_size)

    # pass that batch for training
    x, y = Variable(torch.from_numpy(batch_x)), Variable(torch.from_numpy(batch_y), requires_grad=False)
    pred = model(x)

    # get loss
    loss = loss_fn(pred, y)

    # perform backpropagation
    loss.backward()
    optimizer.step()
    avg_cost += loss.data[0]/total_batch

  print(epoch, avg_cost)</pre>
<pre># get training accuracy
x, y = Variable(torch.from_numpy(preproc(train_x))), Variable(torch.from_numpy(train_y), requires_grad=False)
