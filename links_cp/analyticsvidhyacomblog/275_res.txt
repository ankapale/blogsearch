<h2>Introduction</h2>
<p>What differentiates the best data scientists from others? It is their focus on application of data science. The best data scientists I know of, see data science and its application every where they look. They look at this world as an outcome of flow of data and information.</p>
<p>On the other hand, most beginners often ask the question &#8211; how do we apply our learning on real life problems?</p>
<p>In this post (and another one following this), I have picked up a real life dataset (Stock Markets in India) and showed how I would use this data to come out with useful insights.</p>
<p>I hope that you will find this useful. The idea is show the vast opportunities present in data science in a simple yet powerful manner. If you can think of more examples like this &#8211; let me know in comments below!</p>
<p>For the best results, I would strongly recommend to build the application yourself as you follow the tutorial.</p>
<p>&nbsp;</p>
<h2>Objective of this Tutorial</h2>
<p>In this article, we will analyze stock market in banking segment based on the bank stocks which are listed in <a href="https://www.nseindia.com/live_market/dynaContent/live_watch/pre_open_market/pre_open_market.htm">NSE India</a>. Our objective is to find the trends (Seasonal or cyclic) in banking stocks.</p>
<p>In our comparative analysis we will use several packages and the primary focus will be on <em>tidy verse</em> package. The emphasis will be given on grouping with the help of tibble dataframe from <em>tidy verse</em> package. This will help to perform similar operation on multiple groups at a time, hence reducing the code length and computational time.</p>
<p>This article also focuses on API Key, database code search using quandl, and finally how to directly download the data from R Console.</p>
<p>So lets get started!</p>
<p>Note: The code that has been mentioned below is to be run on the R command line for best results.</p>

<h2>Getting Started with Comparative Analysis</h2>
<h3>Creating the Dataset</h3>
<p>We will be using Quandl is online repository for the core financial, macroeconomic statistics and forex. Quandl has a vast collection of free and open data collected from a variety of organizations: central banks, governments, multinational organizations and more. You can use it without payment and with few restrictions.</p>
<p>Both Free and Premium data are available. Authenticated Free users have a limit of 300 calls per 10 seconds, 2,000 calls per 10 minutes and a limit of 50,000 calls per day. Premium data subscribers have a limit of 5,000 calls per 10 minutes and a limit of 720,000 calls per day.</p>
<p>We will use this online repository to get our data using Quandl package directly from the R Console. Quandl package directly interacts with the Quandl API to offer data in a number of formats usable in R, downloading a zip with all data from a Quandl database, and the ability to search.</p>
<p>For More information on Quandl Package, please visit this <a href="https://cran.r-project.org/web/packages/Quandl/Quandl.pdf" target="_blank" rel="nofollow noopener">page.</a></p>
<p>To get started with Quandl, create an account and get the quandl API key. Please click <a href="https://www.quandl.com/" target="_blank" rel="nofollow noopener">here</a> to create an account. Then click on the Login button provided on the top right corner of the screen. Once the registration is complete, please click <a href="https://www.quandl.com/account/api" target="_blank" rel="nofollow noopener">here</a> to get the API Key.</p>
<p>In our analysis, we have selected following banks</p>
<ul>
<li>ICICI</li>
<li>BOB</li>
<li>CANARA</li>
<li>AXIS</li>
<li>SBI</li>
<li>PNB</li>
</ul>
<p>We have selected these banks as they are in the price band of Rs 200 to Rs 500. We will use the following codes to get the data into R console.</p>
<pre>Quandl(Code=NSE/,collapse=,start_date=-,type=)</pre>
<p>The parameters we use are as follows:</p>
<div id="required-packages" class="section level3">
<ul>
<li><strong>Code</strong> Dataset code on Quandl specified as a string or an array of strings.</li>
<li><strong>collapse</strong> Collapse frequency of Data.Eg; daily, monthly, weekly, yearly.</li>
<li><strong>start_date</strong> Desired Start Date</li>
<li><strong>type</strong> Type of data returned specified as string. Can be raw, ts, zoo, xts or timeSeries</li>
</ul>
<div id="required-packages" class="section level3">
<div id="code-break-up" class="section level3">
<p>Now we will download the data, add a column Stock for the stock identifier, and then we paste the respective stock name in the downloaded dataset. We will then consolidate all stock data into one Master Data frame for analysis</p>
</div>
</div>
<pre class="r"><code class="r"><span class="comment">## Setup the Quandl Free Account and API Key, Please copy and paste the API key in order to #authenticate</span>

<span class="identifier">Quandl.api_key</span><span class="paren">(</span><span class="string">"&lt;Your-API-Key&gt;"</span><span class="paren">)</span>

<h3>Finding the Density Distribution of Deviation of High Price from Open Price</h3>
<p>Now we will see the density distribution of <b>High Price</b> from <b>Open Price</b> in order to get an understanding that how much price is deviating in either direction (North or South) on weekly basis. This gives us an idea of price range for any stock in intraday trading.</p>
<p>We will use the transmute_tq() function from tidyquant package to compute the weekly prices. Please click <a href="https://cran.r-project.org/web/packages/tidyquant/tidyquant.pdf" target="_blank" rel="nofollow noopener">here</a> to get more information.</p>
<p>For this add a new column with the difference of high and open price using mutate function. Add another new column with the difference of low and open price using mutate function. Calculate the weekly average of differences using tq_transmute() function from tidyverse package. Visualize both density plots with dot distribution on ggplot</p>
<pre><i>## Deviation from High &amp; Low Price</i>
Master_Data_High&lt;-Master_Data%&gt;%mutate(Dev_High=High-Open)
Master_Data_Low&lt;-Master_Data%&gt;%mutate(Dev_Low=Open-Low)

<i>## Computation of weekly average for high Price</i>

Master_Data_High_Week &lt;- Master_Data_High %&gt;%
 tq_transmute(
  select   = Dev_High,
  mutate_fun = apply.weekly,
  FUN    = mean,
  na.rm   = TRUE,
  col_rename = "Dev_High_Mean"
 )

<i>## Computation weekly average for Low Price</i>

Master_Data_Low_Week&lt;-Master_Data_Low%&gt;%
  tq_transmute(
    select = Dev_Low,
    mutate_fun = apply.weekly,
    FUN = mean,
    na.rm = TRUE,
    col_rename = "Dev_Low_Mean"
  )

<i>## Visualisation of density distribution of High Price</i>

High&lt;-Master_Data_High_Week%&gt;%ggplot(aes(x=Dev_High_Mean,color=Stock))+
 geom_dotplot(binwidth=0.50,aes(fill=Stock))+
 xlim(0,10)+
 scale_fill_manual(values=c("#999999", "#E69F00","#CC9933","#99FF00","#CC3399","#FF9933"))+
 labs(title="Distribution of High Price Deviation from Open Price",x="Weekly Mean Deviation")+
 facet_wrap(~Stock,ncol=3,scale="free_y")+
 scale_color_tq(values=c("#999999"))+
 theme_tq()+
 theme(panel.border = element_blank(),
    panel.grid.major = element_line(colour = "grey61", size = 0.5, linetype = "dotted"),
    panel.grid.minor = element_blank(),
    axis.line=element_line(colour="black"),
    plot.title = element_text(hjust = 0.5,size=16,colour="indianred4"))+
 theme(legend.position="none")

<i>## Visualisation of density distribution of Low Price</i>
Low&lt;-Master_Data_Low_Week%&gt;%ggplot(aes(x=Dev_Low_Mean,color=Stock))+
 geom_dotplot(binwidth=0.50,aes(fill=Stock))+
 xlim(0,10)+
 scale_fill_manual(values=c("#999999", "#E69F00","#CC9933","#99FF00","#CC3399","#FF9933"))+
 labs(title="Distribution of Weekly Low Price Deviation from Open Price",x="Weekly Mean Deviation")+
 facet_wrap(~Stock,ncol=3,scale="free_y")+
 scale_color_tq(values=c("#999999"))+

 theme(legend.position="none")

<h3>Observing the Autocorrelation lags</h3>
<p>The lag operator (also known as backshift operator) is a function that shifts (offsets) a time series such that the lagged values are aligned with the actual time series. The lags can be shifted any number of units, which simply controls the length of the backshift.</p>
<p>Here, k is denoted as lag. We will see the lag of 180 days period and see how stocks behave.</p>
<p>These are the steps for Computation</p>
<ul>
<li>Define k lag period</li>
<li>Create columns for lag periods</li>
<li>Group the data by Stock by creating new data frame for lags</li>
<li>Apply lag.xts using tq_mutate() function on the new dataframe</li>
<li>Apply Auto-correaltion</li>
</ul>
<pre>k &lt;- 1:180
col_names &lt;- paste0("lag_", k)

<i>## Only Select Columns "Date" and "Close" from hte master data frame.</i>

Master_Data_lags&lt;-Master_Data%&gt;%
tibble::as_tibble() %&gt;%
group_by(Stock)

Master_Data_lags&lt;-Master_Data_lags%&gt;%select(Date,Close)
<i># Apply lag.xts function using tq_mutate</i>

Master_Data_lags&lt;-Master_Data_lags%&gt;%
tq_mutate(
select = Close,
mutate_fun = lag.xts,
k=1:180,
col_rename=col_names
)

<i># Calculate the autocorrelations and 95% cutoffs</i>

Master_Data_AutoCorrelations&lt;-Master_Data_lags %&gt;%
gather(key = "lag", value = "lag_value", -c(Stock,Date, Close)) %&gt;%
mutate(lag = str_sub(lag, start = 5) %&gt;% as.numeric) %&gt;%
group_by(Stock, lag) %&gt;%
summarize(
cor = cor(x = Close, y = lag_value, use = "pairwise.complete.obs"),
cutoff_upper = 2/(n())^0.5,
cutoff_lower = -2/(n())^0.5
)

<i>## Visualisation of Autocorrelation: ACF Plot</i>

Master_Data_AutoCorrelations %&gt;%
ggplot(aes(x = lag, y = cor, color = Stock, group = Stock)) +
<i>
# Add horizontal line a y=0</i>
geom_hline(yintercept = 0) +

<i># Plot autocorrelations</i>
geom_point(size = 2) +
geom_segment(aes(xend = lag, yend = 0), size = 1) +

<i># Add cutoffs</i>
geom_line(aes(y = cutoff_upper), color = "blue", linetype = 2) +
geom_line(aes(y = cutoff_lower), color = "blue", linetype = 2) +

<i># Add facets</i>
facet_wrap(~ Stock, ncol = 3) +

<i># Aesthetics</i>
expand_limits(y = c(-1, 1)) +
scale_color_tq() +
theme_tq() +
labs(
title = paste0("Tidyverse ACF Plot: Lags ", rlang::expr_text(k)),
x = "Lags"
) +
theme(
legend.position = "none",
axis.text.x = element_text(angle = 45, hjust = 1),
panel.grid.major = element_line(colour = "grey61", size = 0.5, linetype = "dotted"),
plot.title = element_text(hjust = 0.5,size=18,colour="indianred4")
)</pre>
