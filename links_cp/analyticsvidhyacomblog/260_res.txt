<div class="text-content">
<div id="ban3_av"></div><h1>Introduction</h1>
<p style="text-align: justify;">If you have spent some time in machine learning and data science, you would have definitely come across imbalanced class distribution. This is a scenario where the number of observations belonging to one class is significantly lower than those belonging to the other classes.</p>
<p style="text-align: justify;">This problem is predominant in scenarios where anomaly detection is crucial like electricity pilferage, fraudulent transactions in banks, identification of rare diseases, etc. In this situation, the predictive model developed using conventional machine learning algorithms could be biased and inaccurate.</p>
<p style="text-align: justify;">This happens because Machine Learning Algorithms are usually designed to improve accuracy by reducing the error. Thus, they do not take into account the class distribution / proportion or balance of classes.</p>
<p style="text-align: justify;">This guide describes various approaches for solving such class imbalance problems using various sampling techniques. We also weigh each technique for its pros and cons. Finally, I reveal an approach using which you can create a balanced class distribution and apply ensemble learning technique designed especially for this purpose.</p>
<p>&nbsp;</p>
<h2>Table of Content</h2>
<ol>
<li>Challenges faced with Imbalanced datasets</li>
<li>Approach to handling Imbalanced Datasets</li>
<li>Illustrative Example</li>
<li>Conclusion</li>
</ol>
<p><a href="https://www.analyticsvidhya.com/datafest-2017/"><img class="aligncenter wp-image-34148 size-full" src="https://s3-ap-south-1.amazonaws.com/av-blog-media/wp-content/uploads/2017/03/22064455/Stripbanner_DF_720.png" alt="" width="720" height="90" srcset="https://s3-ap-south-1.amazonaws.com/av-blog-media/wp-content/uploads/2017/03/22064455/Stripbanner_DF_720.png 720w, https://s3-ap-south-1.amazonaws.com/av-blog-media/wp-content/uploads/2017/03/22064455/Stripbanner_DF_720-300x38.png 300w" sizes="(max-width: 720px) 100vw, 720px" /></a></p>
<h2>1. Challenges faced with Imbalanced datasets</h2>
<p style="text-align: justify;">One of the main challenges faced by the utility industry today is electricity theft. Electricity theft is the third largest form of theft worldwide. Utility companies are increasingly turning towards advanced analytics and machine learning algorithms to identify consumption patterns that indicate theft.</p>
<p style="text-align: justify;">However, one of the biggest stumbling blocks is the humongous data and its distribution. Fraudulent transactions are significantly lower than normal healthy transactions i.e. accounting it to around 1-2 % of the total number of observations. The ask is to improve identification of the rare minority class as opposed to achieving higher overall accuracy.</p>
<p style="text-align: justify;">Machine Learning algorithms tend to produce unsatisfactory classifiers when faced with imbalanced datasets. For any imbalanced data set, if the event to be predicted belongs to the minority class and the event rate is less than 5%, it is usually referred to as a rare event.</p>
<h3 style="text-align: justify;"></h3>
<h3 style="text-align: justify;">Example of imbalanced classes</h3>
<p style="text-align: justify;">Let&#8217;s understand this with the help of an example.</p>
<p style="text-align: justify;"><strong>Ex:</strong> In an utilities fraud detection data set you have the following data:</p>
<p>Total Observations = 1000</p>
<p>Fraudulent  Observations = 20</p>
<p>Non Fraudulent Observations = 980</p>
<p>Event Rate= 2 %</p>
<p style="text-align: justify;">The main question faced during data analysis is <em>&#8211; </em>How to get a balanced dataset by getting a decent number of samples for these anomalies given the rare occurrence for some them?</p>
<h3 style="text-align: justify;"></h3>
<h3 style="text-align: justify;">Challenges with standard Machine learning techniques</h3>
<p style="text-align: justify;">The conventional model evaluation methods do not accurately measure model performance when faced with imbalanced datasets.</p>
<p style="text-align: justify;">Standard classifier algorithms like Decision Tree and Logistic Regression have a bias towards classes which have number of instances. They tend to only predict the majority class data. The features of the minority class are treated as noise and are often ignored. Thus, there is a high probability of misclassification of the minority class as compared to the majority class.</p>
<p style="text-align: justify;">Evaluation of a classification algorithm performance is measured by the Confusion Matrix which contains information about the actual and the predicted class.</p>
<p style="text-align: center;"><em><img class="aligncenter wp-image-33968 size-full" src="https://s3-ap-south-1.amazonaws.com/av-blog-media/wp-content/uploads/2017/03/16142827/ICP1.png" alt="" width="471" height="171" srcset="https://s3-ap-south-1.amazonaws.com/av-blog-media/wp-content/uploads/2017/03/16142827/ICP1.png 471w, https://s3-ap-south-1.amazonaws.com/av-blog-media/wp-content/uploads/2017/03/16142827/ICP1-300x109.png 300w" sizes="(max-width: 471px) 100vw, 471px" /><br />
</em><strong>Accuracy of a model = (TP+TN) / (TP+FN+FP+TN)</strong></p>
<p style="text-align: justify;">However, while working in an imbalanced domain accuracy is not an appropriate measure to evaluate model performance<strong>. </strong>For eg: A classifier which achieves an accuracy of 98 % with an event rate of 2 % is not accurate, if it classifies all instances as the majority class. And eliminates the 2 % minority class observations as noise.</p>
<h3 style="text-align: justify;"></h3>
<h3 style="text-align: justify;">Examples of imbalanced classes</h3>
<p style="text-align: justify;">Thus, to sum it up, while trying to resolve specific business challenges with imbalanced data sets, the classifiers produced by standard machine learning algorithms might not give accurate results. Apart from fraudulent transactions, other examples of a common business problem with imbalanced dataset are:</p>
<ul>
<li style="text-align: justify;">Datasets to identify customer churn where a vast majority of customers will continue using the service. Specifically, Telecommunication companies where Churn Rate is lower than 2 %.</li>
<li>Data sets to identify rare diseases in medical diagnostics etc.</li>
<li>Natural Disaster like Earthquakes</li>
</ul>
<h3 style="text-align: justify;"></h3>
<h3 style="text-align: justify;">Dataset used</h3>
<p style="text-align: justify;">In this article, we will illustrate the various techniques to train a model to perform well against highly imbalanced datasets. And accurately predict rare events using the following fraud detection dataset:</p>
<p>Total Observations = 1000</p>
<p>Fraudulent   Observations =20</p>
<p>Non-Fraudulent Observations = 980</p>
<p>Event Rate= 2 %</p>
<p>Fraud Indicator = 0 for Non-Fraud Instances</p>
<p>Fraud Indicator = 1 for Fraud</p>
<p>&nbsp;</p>
<h2>2. Approach to handling Imbalanced Datasets</h2>
<h3>2.1 Data Level approach: Resampling Techniques</h3>
<p style="text-align: justify;">Dealing with imbalanced datasets entails strategies such as improving classification algorithms or balancing classes in the training data (data preprocessing) before providing the data as input to the machine learning algorithm. The later technique is preferred as it has wider application.</p>
<p style="text-align: justify;">The main objective of balancing classes is to either increasing the frequency of the minority class or decreasing the frequency of the majority class. This is done in order to obtain approximately the same number of instances for both the classes. Let us look at a few resampling techniques:</p>
<p>&nbsp;</p>
<h4>2.1.1  Random Under-Sampling</h4>
<p>Random Undersampling aims to balance class distribution by randomly eliminating majority class examples.  This is done until the majority and minority class instances are balanced out.</p>
<p>Total Observations = 1000</p>
<p>Fraudulent   Observations =20</p>
<p>Non Fraudulent Observations = 980</p>
<p>Event Rate= 2 %</p>
<p style="text-align: justify;">In this case we are taking 10 % samples without replacement from Non Fraud instances.  And combining them with Fraud instances.</p>
<p>Non Fraudulent Observations after random under sampling = 10 % of 980 =98</p>
<p>Total Observations after combining them with Fraudulent observations = 20+98=118</p>
<p>Event Rate for the new dataset after under sampling = 20/118 = 17%</p>
<p>&nbsp;</p>
<ul>
<li><strong>Advantages</strong>
<ul>
<li style="text-align: justify;">It can help improve run time and storage problems by reducing the number of training data samples when the training data set is huge.</li>
</ul>
</li>
</ul>
<ul>
<li><strong>Disadvantages</strong>
<ul>
<li>It can discard potentially useful information which could be important for building rule classifiers.</li>
<li style="text-align: justify;">The sample chosen by random under sampling may be a biased sample. And it will not be an accurate representative of the population. Thereby, resulting in inaccurate results with the actual test data set.</li>
</ul>
</li>
</ul>
<p>&nbsp;</p>
<h4>2.1.2  Random Over-Sampling</h4>
<p style="text-align: justify;">Over-Sampling increases the number of instances in the minority class by randomly replicating them in order to present a higher representation of the minority class in the sample.</p>
<p>Total Observations = 1000</p>
<p>Fraudulent   Observations =20</p>
<p>Non Fraudulent Observations = 980</p>
<p>Event Rate= 2 %</p>
<p>In this case we are replicating 20 fraud observations   20 times.</p>
<p>Non Fraudulent Observations =980</p>
<p>Fraudulent Observations after replicating the minority class observations= 400</p>
<p>Total Observations in the new data set after oversampling=1380</p>
<p>Event Rate for the new data set after under sampling= 400/1380 = 29 %</p>
<ul>
<li><strong>Advantages</strong>
<ul>
<li>Unlike under sampling this method leads to no information loss.</li>
<li>Outperforms under sampling</li>
</ul>
</li>
</ul>
<ul>
<li><strong>Disadvantages</strong>
<ul>
<li>It increases the likelihood of overfitting since it replicates the minority class events.</li>
</ul>
</li>
</ul>
<p><strong> </strong></p>
<h4>2.1.3  Cluster-Based Over Sampling</h4>

<h4>2.1.4  Informed Over Sampling: Synthetic Minority Over-sampling Technique</h4>
<p style="text-align: justify;">This technique is followed to avoid overfitting which occurs when exact replicas of minority instances are added to the main dataset. A subset of data is taken from the minority class as an example and then new synthetic similar instances are created. These synthetic instances are then added to the original dataset. The new dataset is used as a sample to train the classification models.</p>

<li><strong>Advantages</strong>

<li>No loss of useful information</li>

<li>SMOTE is not very effective for high dimensional data</li>

<h4>2.1.5  Modified synthetic minority oversampling technique (MSMOTE)</h4>
<p style="text-align: justify;">It is a modified version of SMOTE. SMOTE does not consider the underlying distribution of the minority class and latent noises in the dataset. To improve the performance of SMOTE a modified method MSMOTE is used.</p>
<p style="text-align: justify;">This algorithm classifies the samples of minority classes into 3 distinct groups &#8211; Security/Safe samples, Border samples, and latent nose samples. This is done by calculating the distances among samples of the minority class and samples of the training data.</p>
<p style="text-align: justify;">Security samples are those data points which can improve the performance of a classifier. While on the other hand, noise are the data points which can reduce the performance of the classifier.  The ones which are difficult to categorize into any of the two are classified as border samples.</p>
<p style="text-align: justify;">While the basic flow of MSOMTE is the same as that of SMOTE (discussed in the previous section).  In MSMOTE the strategy of selecting nearest neighbors is different from SMOTE<strong>.</strong> The algorithm randomly selects a data point from the k nearest neighbors for the security sample, selects the nearest neighbor from the border samples and does nothing for latent noise.</p>

<h3>2.2 Algorithmic Ensemble Techniques</h3>
<p style="text-align: justify;">The above section, deals with handling imbalanced data by resampling original data to provide balanced classes. In this section, we are going to look at an alternate approach i.e.  Modifying existing classification algorithms to make them appropriate for imbalanced data sets.</p>
<p style="text-align: justify;">The main objective of ensemble methodology is to improve the performance of single classifiers. The approach involves constructing several two stage classifiers from the original data and then aggregate their predictions.<strong><br />
<img class="aligncenter wp-image-33971 size-full" src="https://s3-ap-south-1.amazonaws.com/av-blog-media/wp-content/uploads/2017/03/16142904/ICP4.png" alt="" width="626" height="277" srcset="https://s3-ap-south-1.amazonaws.com/av-blog-media/wp-content/uploads/2017/03/16142904/ICP4.png 626w, https://s3-ap-south-1.amazonaws.com/av-blog-media/wp-content/uploads/2017/03/16142904/ICP4-300x133.png 300w" sizes="(max-width: 626px) 100vw, 626px" /> </strong></p>

<h4>2.2.1. Bagging Based</h4>
<p style="text-align: justify;">Bagging is an abbreviation of Bootstrap Aggregating. The conventional bagging algorithm involves generating &#8216;n&#8217; different bootstrap training samples with replacement. And training the algorithm on each bootstrapped algorithm separately and then aggregating the predictions at the end.</p>
<p style="text-align: justify;">Bagging is used for reducing Overfitting in order to create strong learners for generating accurate predictions. Unlike boosting, bagging allows replacement in the bootstrapped sample.</p>
<p style="text-align: justify;"><img class="aligncenter wp-image-33972 size-full" src="https://s3-ap-south-1.amazonaws.com/av-blog-media/wp-content/uploads/2017/03/16142914/ICP5.png" alt="" width="591" height="414" srcset="https://s3-ap-south-1.amazonaws.com/av-blog-media/wp-content/uploads/2017/03/16142914/ICP5.png 591w, https://s3-ap-south-1.amazonaws.com/av-blog-media/wp-content/uploads/2017/03/16142914/ICP5-300x210.png 300w" sizes="(max-width: 591px) 100vw, 591px" /></p>
<p>&nbsp;</p>
<p style="text-align: center;"><strong>                                     Figure 4:  Approach to Bagging Methodology </strong></p>
<p>Total Observations = 1000</p>
<p>Fraudulent   Observations =20</p>
<p>Non Fraudulent Observations = 980</p>
<p>Event Rate= 2 %</p>
<p style="text-align: justify;">There are 10 bootstrapped samples chosen from the population with replacement. Each sample contains 200 observations. And each sample is different from the original dataset but resembles the dataset in distribution &amp; variability<strong>. </strong></p>
<p style="text-align: justify;">The machine learning algorithms like logistic regression, neural networks, decision tree  are fitted to each bootstrapped sample of 200 observations. And the Classifiers c1, c2…c10 are aggregated to produce a compound classifier.  This ensemble methodology produces a stronger compound classifier since it combines the results of individual classifiers to come up with an improved one.</p>
<p>&nbsp;</p>
<ul>
<li><strong>Advantages</strong>
<ul>
<li>Improves stability &amp; accuracy of machine learning algorithms</li>
<li>Reduces variance</li>
<li>Overcomes overfitting</li>
<li>Improved misclassification rate of the bagged classifier</li>
<li>In noisy data environments bagging outperforms boosting</li>
</ul>
</li>
</ul>
<p>&nbsp;</p>
<ul>
<li><strong>Disadvantages</strong>
<ul>
<li>Bagging works only if the base classifiers are not bad to begin with. Bagging bad classifiers can further degrade performance</li>
</ul>
</li>
</ul>

<h4>2.2.2. Boosting-Based</h4>
<p style="text-align: justify;">Boosting is an ensemble technique to combine weak learners to create a strong learner that can make accurate predictions. Boosting starts out with a base classifier / weak classifier that is prepared on the training data.</p>
<p>What are base learners / weak classifiers?</p>
<p style="text-align: justify;">The base learners / Classifiers are weak learners i.e. the prediction accuracy is only slightly better than average. A classifier learning algorithm is said to be weak when small changes in data induce big changes in the classification model.</p>
<p style="text-align: justify;">In the next iteration, the new classifier focuses on or places more weight to those cases which were incorrectly classified in the last round.</p>
<p style="text-align: justify;"><img class="aligncenter wp-image-33973 size-full" src="https://s3-ap-south-1.amazonaws.com/av-blog-media/wp-content/uploads/2017/03/16142924/ICP6.png" alt="" width="693" height="452" srcset="https://s3-ap-south-1.amazonaws.com/av-blog-media/wp-content/uploads/2017/03/16142924/ICP6.png 693w, https://s3-ap-south-1.amazonaws.com/av-blog-media/wp-content/uploads/2017/03/16142924/ICP6-300x196.png 300w, https://s3-ap-south-1.amazonaws.com/av-blog-media/wp-content/uploads/2017/03/16142924/ICP6-258x169.png 258w" sizes="(max-width: 693px) 100vw, 693px" /></p>
<p>&nbsp;</p>
<p style="text-align: center;"><strong>Figure 5: Approach to Boosting Methodologies </strong></p>
<p><strong> </strong></p>
<h4>2.2.2.1. Adaptive Boosting- Ada Boost</h4>
<p style="text-align: justify;">Ada Boost is the first original boosting technique which creates a highly accurate prediction rule by combining many weak and inaccurate rules.  Each classifier is serially trained with the goal of correctly classifying examples in every round that were incorrectly classified in the previous round.</p>
<p>For a learned classifier to make strong predictions it should follow the following three conditions:</p>
<ul>
<li>The rules should be simple</li>
<li>Classifier should have been trained on sufficient number of training examples</li>
<li>The Classifier should have low training error for the training instances</li>
</ul>
<p style="text-align: justify;">Each of the weak hypothesis has an accuracy slightly better than random guessing i.e. Error Term € (t) should be slightly more than ½-β where β &gt;0. This is the fundamental assumption of this boosting algorithm which can produce a final hypothesis with a small error</p>
<p style="text-align: justify;">After each round, it gives more focus to examples that are harder to classify.  The quantity of focus is measured by a weight, which initially is equal for all instances. After each iteration, the weights of misclassified instances are increased and the weights of correctly classified instances are decreased.</p>
<p>&nbsp;</p>
<p style="text-align: center;"><strong><img class="aligncenter wp-image-33974 size-full" src="https://s3-ap-south-1.amazonaws.com/av-blog-media/wp-content/uploads/2017/03/16142934/ICP7.png" alt="" width="703" height="615" srcset="https://s3-ap-south-1.amazonaws.com/av-blog-media/wp-content/uploads/2017/03/16142934/ICP7.png 703w, https://s3-ap-south-1.amazonaws.com/av-blog-media/wp-content/uploads/2017/03/16142934/ICP7-300x262.png 300w" sizes="(max-width: 703px) 100vw, 703px" />                                       </strong></p>
<p style="text-align: center;"><strong>Figure 6:  Approach to Adaptive Boosting </strong></p>
<p style="text-align: justify;">For example in a data set containing 1000 observations out of which 20 are labelled fraudulent. Equal weights W1 are assigned to all observations and the base classifier accurately classifies 400 observations.</p>
<p style="text-align: justify;">Weight of each of the 600 misclassified observations is increased to w2 and weight of each of the correctly classified observations is reduced to w3.</p>
<p style="text-align: justify;">In each iteration, these updated weighted observations are fed to the weak classifier to improve its performance. This process continues till the misclassification rate significantly decreases thereby resulting in a strong classifier.</p>
<p>&nbsp;</p>
<ul>
<li><strong>Advantages</strong>
<ol>
<li>Very Simple to implement</li>
<li>Good generalization- suited for any kind of classification problem ü Not prone to overfitting</li>
</ol>
</li>
</ul>
<p>&nbsp;</p>
<ul>
<li><strong>Disadvantages</strong>
<ol>
<li>Sensitive to noisy data and outliers</li>
</ol>
</li>
</ul>

<h4>2.2.2.2  Gradient Tree Boosting</h4>
<p style="text-align: justify;">In Gradient Boosting many models are trained sequentially. It is a numerical optimization algorithm where each model minimizes the loss function, <strong>y = ax+b+e</strong>, using the Gradient Descent Method.</p>
<p>Decision Trees are used as weak learners in Gradient Boosting.</p>
<p style="text-align: justify;">While both Adaboost and Gradient Boosting work on weak learners / classifiers. And try to boost them into a strong learner, there are some fundamental differences in the two methodologies. Adaboost either requires the users to specify a set of weak learners  or randomly generates the weak learners before the actual learning process. The weight of each learner is adjusted at every step depending on whether it predicts a sample correctly.</p>
<p style="text-align: justify;">On the other hand, Gradient Boosting builds the first learner on the training dataset to predict the samples, calculates the loss (Difference between real value and output of the first learner). And use this loss to build an improved learner in the second stage.</p>
<p style="text-align: justify;">At every step, the residual of the loss function is calculated using the Gradient Descent Method and the new residual becomes a target variable for the subsequent iteration.</p>
<p>Gradient Boosting can be done using the Gradient Boosting Node in SAS Miner and GBM package in R</p>
<p><img class="aligncenter wp-image-33975 size-full" src="https://s3-ap-south-1.amazonaws.com/av-blog-media/wp-content/uploads/2017/03/16142945/ICP8.png" alt="" width="539" height="605" srcset="https://s3-ap-south-1.amazonaws.com/av-blog-media/wp-content/uploads/2017/03/16142945/ICP8.png 539w, https://s3-ap-south-1.amazonaws.com/av-blog-media/wp-content/uploads/2017/03/16142945/ICP8-267x300.png 267w" sizes="(max-width: 539px) 100vw, 539px" /></p>
<p>&nbsp;</p>
<p style="text-align: center;"><strong>       Figure 7:  Approach to Gradient Boosting </strong></p>
<p style="text-align: justify;">For example: In a training data set containing 1000 observations out of which 20 are labelled fraudulent an initial base classifier. Target Variable Fraud =1 for fraudulent transactions and Fraud=0 for not fraud transactions.</p>
<p style="text-align: justify;">For eg: Decision tree is fitted which accurately classifying only 5 observations as Fraudulent observations. A differentiable loss function is calculated based on the difference between the actual output and the predicted output of this step.  The residual of the loss function is the target variable (F1) for the next iteration.</p>
<p style="text-align: justify;">Similarly, this algorithm internally calculates the loss function, updates the target at every stage and comes up with an improved classifier as compared to the initial classifier.</p>
<p>&nbsp;</p>
<ul>
<li><strong>Disadvantages</strong>
<ul>
<li>Gradient Boosted trees are harder to fit than random forests</li>
<li style="text-align: justify;">Gradient Boosting Algorithms generally have 3 parameters which can be fine-tuned, Shrinkage parameter, depth of the tree, the number of trees. Proper training of each of these parameters is needed for a good fit. If parameters are not tuned correctly it may result in over-fitting.</li>
</ul>
</li>
</ul>

<p><strong>2.2.2.3</strong><strong> XG Boost </strong></p>
<p style="text-align: justify;">XGBoost (Extreme Gradient Boosting) is an advanced and more efficient implementation of Gradient Boosting Algorithm discussed in the previous section.</p>
<p>Advantages over Other Boosting Techniques</p>
<ul>
<li style="text-align: justify;">It is 10 times faster than the normal Gradient Boosting as it implements parallel processing. It is highly flexible as users can define custom optimization objectives and evaluation criteria, has an inbuilt mechanism to handle missing values.</li>
<li style="text-align: justify;">Unlike gradient boosting which stops splitting a node as soon as it encounters a negative loss, XG Boost splits up to the maximum depth specified and prunes the tree backward and removes splits beyond which there is an only negative loss.</li>
</ul>
<p>Extreme gradient boosting can be done using the XGBoost package in R and Python</p>

<h2>4. Conclusion</h2>
<p style="text-align: justify;">When faced with imbalanced data sets there is no one stop solution to improve the accuracy of the prediction model.  One may need to try out multiple methods to figure out the best-suited sampling techniques for the dataset. In most cases, synthetic techniques like SMOTE and MSMOTE will outperform the conventional oversampling and undersampling methods.</p>
<p>For better results, one can use synthetic sampling methods like SMOTE and MSMOTE along with advanced boosting methods like Gradient boosting and XG Boost.</p>
<p style="text-align: justify;">One of the advanced bagging techniques commonly used to counter the imbalanced dataset problem is SMOTE bagging. It follows an entirely different approach from conventional bagging to create each Bag/Bootstrap. It generates the positive instances by the SMOTE Algorithm by setting a SMOTE resampling rate in each iteration. The set of negative instances is bootstrapped in each iteration.</p>
<p style="text-align: justify;">Depending on the characteristics of the imbalanced data set, the most effective techniques will vary. Relevant evaluation parameters should be considered during the model comparison.</p>
<p style="text-align: justify;">While comparing multiple prediction models built through an exhaustive combination of the above-mentioned techniques Lift &amp; Area under the ROC Curve will be instrumental in determining which model is superior to the others.</p>
