<div id="ban3_av"></div><h2><span style="font-weight: 400;">Introduction</span></h2>
<p><em>I still remember my first encounter with a Click prediction problem. Before this, I had been learning data science and I was feeling good about my progress. I had started to build my confidence in ML hackathons and I was determined to do well in several challenges.</em></p>
<p><em>In order to do well, I had even procured a machine with 16 GB RAM and i7 processor. But the first look at the dataset gave me jitters. The data when unzipped was over 50 GB &#8211; I had no clue how to predict a click on such a dataset. Thankfully Factorization machines came to my rescue.</em></p>
<p><span style="font-weight: 400;"><span class="c1">Anyone who has worked on a Click Prediction problem or Recommendation systems would have faced a similar situation. Since the datasets are huge, doing predictions for these datasets becomes challenging with limited computation resources. </span></span></p>
<p><span style="font-weight: 400;"><span class="c1">However, in most cases these datasets are sparse (only a few variables for each training example are non zero) due to which there are several features which are not important for prediction, this is where factorization helps to extract the most important latent or hidden features from the existing raw ones.</span></span></p>
<p><span style="font-weight: 400;"><span class="c1">Factorization helps in representing approximately the same relationship between the target and predictors using a lower dimension dense matrix. In this article, I discuss Factorization Machines(FM) and Field Aware Factorization Machines(FFM) which allows us to take advantage of factorization in a regression/classification problem with an implementation using python.</span></span></p>
<p>&nbsp;</p>

<p>&nbsp;</p>
<p class="c2"><span class="c1">We observe from the table above that some of the ratings are missing and we would like to devise a method to predict these missing ratings. The intuition behind using matrix factorization to solve this problem is that there should be some latent features that determines how a user rates a movie. For example &#8211; users A and B would rate an Al Pacino movie highly if both of them are fans of actor Al Pacino, here a preference towards a particular actor would be a hidden feature since we are not explicitly including it in the rating matrix.</span></p>
<p class="c2">Suppose we want to compute K hidden or latent features.<span style="font-weight: 400;"> Our task is to find out the matrices P(U x K) and Q(D x K) (U &#8211; Users, D &#8211; Movies) such that P x Q</span><span style="font-weight: 400;"><sup>T</sup></span><span style="font-weight: 400;"> approximates R which is the rating matrix.</span> <img class="aligncenter wp-image-40135 " src="https://s3-ap-south-1.amazonaws.com/av-blog-media/wp-content/uploads/2017/12/20174727/user_movie.jpg" alt="" width="420" height="186" srcset="https://s3-ap-south-1.amazonaws.com/av-blog-media/wp-content/uploads/2017/12/20174727/user_movie.jpg 713w, https://s3-ap-south-1.amazonaws.com/av-blog-media/wp-content/uploads/2017/12/20174727/user_movie-300x133.jpg 300w" sizes="(max-width: 420px) 100vw, 420px" /> <span style="font-weight: 400;">Now, each row of P will represent strength of association between user and the feature while each row of Q represents the same strength w.r.t. the movie. To get the rating of a movie d</span><span style="font-weight: 400;"><sub>j</sub></span><span style="font-weight: 400;"> rated by user u</span><span style="font-weight: 400;"><sub>i</sub></span><span style="font-weight: 400;">, we can calculate the dot product of 2 vectors corresponding to u</span><span style="font-weight: 400;"><sub>i</sub></span><span style="font-weight: 400;"> and d</span><span style="font-weight: 400;"><sub>j</sub></span> <img class="wp-image-40147 aligncenter" src="https://s3-ap-south-1.amazonaws.com/av-blog-media/wp-content/uploads/2017/12/20190955/rating_eqn.jpg" alt="" width="112" height="36" /> <span style="font-weight: 400;">All we need to do now is calculate P and Q matrices. <span class="c3"><span class="c1">We use gradient descent algorithm for doing this. The objective is to minimize the squared error between the actual rating and the one estimated by P and Q. The squared error is given by the following equation</span>.</span></span></p>
<p><img class="aligncenter" title="" src="https://lh6.googleusercontent.com/XiAvQAeN307wZkPtseFLWUEjTsX09uqRJK0x_0JCirTfkq7FevkwiUFJCuVk9yOnc1-FUOMJOx5Y8Td_kH-D0bxm3ctXFpkNRxEovnI0EH8dEs8OwNbgOeXWU-d8mr-VHH4B2t7W" alt="" width="365" height="40" /></p>
<p><span class="c3">Now, we need to define an update rule for p<sub>ik</sub> and q<sub>kj</sub>. The update rule in gradient descent is defined by the gradient of the error to be minimized. </span></p>

<p>When we talk about FMs or FFMs, each column (Publisher, Advertiser&#8230;) in the dataset would be referred to as a <strong>field </strong>and each value (ESPN, Nike&#8230;.) would be referred to as a <strong>feature</strong>.</p>
<p><span style="font-weight: 400;">A linear or a logistic modeling technique is great and does well in a variety of problems but the drawback is that the model only learns the effect of all variables or features individually rather than in combination.</span></p>

<p><span style="font-weight: 400;">Where </span><span style="font-weight: 400;">w</span><span style="font-weight: 400;"><sub>0</sub></span><span style="font-weight: 400;">, </span><span style="font-weight: 400;">w</span><span style="font-weight: 400;"><sub>ESPN</sub></span><span style="font-weight: 400;"> etc. represent the parameters and </span><span style="font-weight: 400;">x</span><span style="font-weight: 400;"><sub>ESPN</sub></span><span style="font-weight: 400;">, </span><span style="font-weight: 400;">x</span><span style="font-weight: 400;"><sub>Nike</sub></span><span style="font-weight: 400;">represent the individual features in the dataset. <p><span style="font-weight: 400;">This can also be referred to as Poly2 model as we are only considering combination of 2 features for a term.</span></p>
<ul>
<li><span style="font-weight: 400;">The problem with this is that even for a medium sized dataset, we have a huge model that has terrible implications for both the amount of memory needed to store the model and the time it takes to train the model. </span></li>
<li><span style="font-weight: 400;">Secondly, for a sparse dataset this technique will not do well to learn all the weights or parameters reliably i.e. we will not have enough training examples for each pair of features in order for each weight to be reliable.</span></li>
</ul>

<h3>FM to the rescue</h3>
<p><span style="font-weight: 400;">FM solves the problem of considering pairwise feature interactions. It allows us to train, based on reliable information (latent features) from every pairwise combination of features in the model. FM also allows us to do this in an efficient way both in terms of time and space complexity.</span> It models pairwise feature interactions as the dot product of low dimensional vectors(length = k). This is illustrated with the following equation for a degree = 2 factorization machine: <img class="aligncenter wp-image-40161 " src="https://s3-ap-south-1.amazonaws.com/av-blog-media/wp-content/uploads/2017/12/20232823/fm_eqn1.jpg" alt="" width="799" height="50" srcset="https://s3-ap-south-1.amazonaws.com/av-blog-media/wp-content/uploads/2017/12/20232823/fm_eqn1.jpg 1156w, https://s3-ap-south-1.amazonaws.com/av-blog-media/wp-content/uploads/2017/12/20232823/fm_eqn1-300x19.jpg 300w, https://s3-ap-south-1.amazonaws.com/av-blog-media/wp-content/uploads/2017/12/20232823/fm_eqn1-768x48.jpg 768w, https://s3-ap-south-1.amazonaws.com/av-blog-media/wp-content/uploads/2017/12/20232823/fm_eqn1-850x53.jpg 850w" sizes="(max-width: 799px) 100vw, 799px" /> Each parameter in FMs (k=3) can be described as follows: <img class="aligncenter wp-image-40163 " src="https://s3-ap-south-1.amazonaws.com/av-blog-media/wp-content/uploads/2017/12/20233722/fm_eqn_3.jpg" alt="" width="624" height="48" srcset="https://s3-ap-south-1.amazonaws.com/av-blog-media/wp-content/uploads/2017/12/20233722/fm_eqn_3.jpg 907w, https://s3-ap-south-1.amazonaws.com/av-blog-media/wp-content/uploads/2017/12/20233722/fm_eqn_3-300x23.jpg 300w, https://s3-ap-south-1.amazonaws.com/av-blog-media/wp-content/uploads/2017/12/20233722/fm_eqn_3-768x58.jpg 768w, https://s3-ap-south-1.amazonaws.com/av-blog-media/wp-content/uploads/2017/12/20233722/fm_eqn_3-850x65.jpg 850w" sizes="(max-width: 624px) 100vw, 624px" /> <span style="font-weight: 400;">Here, for each term we have calculated the dot product of the 2 latent factors of size 3 corresponding to the 2 features.</span></p>
<p class="c2">From a modeling perspective, this is powerful because each feature ends up transformed to a space where similar features are embedded near one another. In simple words, the dot product basically represents similarity of the hidden features and it is higher when the features are in the neighborhood.</p>
<p class="c2"><span class="c1"><img class="aligncenter" title="" src="https://lh6.googleusercontent.com/jiKwNexUqpNwKlFAk9pmyLQiYF7Nt1a3kHlG_DDSdHRCma9Rj_la1xOyeqOsI-4pVefTfYdpASFzu23JphaDgNIzwTaBo_Go38j3vyRxiOObXWhTHKFaECMAri0MlcZT-KCyNp5K" alt="" /></span></p>
<p><span style="font-weight: 400;"><span class="c1">The cosine function is 1 (maximum) when theta is 0 and decreases to -1 when theta is 180 degrees. It is clear that the similarity is maximum when theta approaches 0.</span></span></p>
<p><span style="font-weight: 400;">Another big advantage of FMs is that we are able to compute the term that models all pairwise interactions in <strong>linear time complexity</strong>using a simple mathematical manipulation to the above equation. If you want to have a look at the exact steps required for this, please refer to the original Factorization Machines research paper at this <a href="https://www.csie.ntu.edu.tw/~b97053/paper/Rendle2010FM.pdf">link</a>.</span></p>

<ul>
<li><span style="font-weight: 400;">There is only one negative training data for the pair (ESPN, Adidas). <p><span class="c1">In order to understand FFMs, we need to realize the meaning of field. Field is typically the broader category which contains a particular feature. In the above training example, the fields are Publisher (P), Advertiser (A) and Gender(G).</span></p>
<ul>
<li><span style="font-weight: 400;">In FMs, every feature has only one latent vector v to learn the latent effect with any other features. Take ESPN as an example, w</span><span style="font-weight: 400;"><sub>ESPN</sub></span><span style="font-weight: 400;"> is used to learn the latent effect with Nike (w</span><span style="font-weight: 400;"><sub>ESPN</sub></span><span style="font-weight: 400;">w</span><span style="font-weight: 400;"><sub>Nike</sub></span><span style="font-weight: 400;">) and Male (w</span><span style="font-weight: 400;"><sub>ESPN</sub></span><span style="font-weight: 400;">.w</span><span style="font-weight: 400;"><sub>Male</sub></span><span style="font-weight: 400;">). </span></li>
<li><span style="font-weight: 400;">However, because ESPN and Male belong to different fields, the latent effects of (ESPN, Nike) and (ESPN, Male) may be different. This is not captured by factorization machines as it will use the same parameters for dot product in both cases. </span></li>
<li><span style="font-weight: 400;"><span class="c1">In FFMs, each feature has several latent vectors. For example, when we consider the interaction term for ESPN and Nike, the hidden feature for ESPN would have the notation w<sub>ESPN,A</sub> where A(Advertiser) represents the field for the feature Nike. Similarly for Gender field a different parameter w<sub>ESPN,G</sub> would be learnt.</span></span></li>
</ul>

<p class="c2"><span class="c1">In case of a categorical field, the feature is uniquely encoded and a value of 1 is assigned to it. In the above figure ESPN is represented by code 1, Nike is represented by code 2 and so on. Each line contains an equivalent training example and is ended by a &#8216;\n&#8217; or a new line character.</span></p>
<ul>
<li>For classification(binary/multiclass), &lt;label&gt; is an integer indicating the class label.</li>
<li>For regression, &lt;label&gt; is the target value which can be any real number.</li>
<li>Labels in the test file are only used to calculate accuracy or errors. If they are unknown, you can just fill the first column with any number.</li>
</ul>
<p class="c2"><span class="c1">Similarly for FFMs, the data needs to be transformed to a libffm format. Here, we also need to encode the field since ffm requires the information of field for learning. The format for the same is:</span></p>
<p class="c2"><span class="c1">&lt;label&gt; &lt;field1&gt;:&lt;feature1&gt;:&lt;value1&gt; &lt;field2&gt;:&lt;feature2&gt;:&lt;value2&gt; &#8230;..</span></p>
<p>&nbsp;</p>
<p><img class="aligncenter wp-image-40143 " src="https://s3-ap-south-1.amazonaws.com/av-blog-media/wp-content/uploads/2017/12/20183838/ffm_format.jpg" alt="" width="472" height="283" srcset="https://s3-ap-south-1.amazonaws.com/av-blog-media/wp-content/uploads/2017/12/20183838/ffm_format.jpg 782w, https://s3-ap-south-1.amazonaws.com/av-blog-media/wp-content/uploads/2017/12/20183838/ffm_format-300x180.jpg 300w, https://s3-ap-south-1.amazonaws.com/av-blog-media/wp-content/uploads/2017/12/20183838/ffm_format-768x462.jpg 768w" sizes="(max-width: 472px) 100vw, 472px" /></p>
<h3 style="text-align: left;"><span style="font-weight: 400;">Important note on numerical features</span></h3>
<p style="text-align: left;"><span style="font-weight: 400;">Numerical features either need to be discretized (transformed to categorical features by breaking the entire range of a particular numerical feature into smaller ranges and label encoding each range separately) and then converted to libffm format as described above. </span></p>
<p style="text-align: left;"><span style="font-weight: 400;">Another possibility is to add a dummy field which is the same as feature value will be numeric feature for that particular row (For example a feature with value 45.3 can be transformed to 1:1:45.3). However, the dummy fields may not be informative because they are merely duplicates of features.</span></p>
<h3>xLearn</h3>
<p style="text-align: left;"><span style="font-weight: 400;">Recently launched xLearn library provides a fast solution to implementing FM and FFM models on a variety of datasets. It is much faster than libfm and libffm libraries and provide a better functionality for model testing and tuning.</span></p>
<p><img class="aligncenter wp-image-40165 " src="https://s3-ap-south-1.amazonaws.com/av-blog-media/wp-content/uploads/2017/12/21000403/xlearn.jpg" alt="" width="354" height="157" srcset="https://s3-ap-south-1.amazonaws.com/av-blog-media/wp-content/uploads/2017/12/21000403/xlearn.jpg 397w, https://s3-ap-south-1.amazonaws.com/av-blog-media/wp-content/uploads/2017/12/21000403/xlearn-300x133.jpg 300w" sizes="(max-width: 354px) 100vw, 354px" /></p>
<p style="text-align: left;"><span style="font-weight: 400;">Here, we will illustrate with an example of FFM for the loan prediction dataset which can be accessed at the <a href="https://datahack.analyticsvidhya.com/contest/practice-problem-loan-prediction-iii/">Loan Predictionpractice problem</a>.</span></p>
<p>The problem to identify the customers segments eligible for loan amount so that they can specifically target these customers based on some demographic and credit history variables.</p>
<p>First step is to import necessary libraries</p>
<pre>import pandas as pd
import xlearn as xl
train = pd.read_csv('train_u6lujuX_CVtuZ9i.csv')
import warnings
warnings.filterwarnings('ignore')</pre>
<p>For simplicity we will just take a few variables here:</p>
<pre>cols = ['Education','ApplicantIncome','Loan_Status','Credit_History']
train_sub = train[cols]
train_sub['Credit_History'].fillna(0, inplace = True)
dict_ls = {'Y':1, 'N':0}
train_sub['Loan_Status'].replace(dict_ls, inplace = True)</pre>
<p>Next we will create a test set for testing the ffm model</p>
<pre>from sklearn.model_selection import train_test_split
X_train, X_test = train_test_split(train_sub, test_size = 0.3, random_state = 5)</pre>
<p><span style="font-weight: 400;">Next, we need to convert the dataset to libffm format which is necessary for xLearn to fit the model. Following function does the job of converting dataset in standard dataframe format to libffm format. </span></p>
<p style="text-align: left;"><span style="font-weight: 400;">df = Dataframe to be converted to ffm format</span></p>
<p style="text-align: left;"><span style="font-weight: 400;">Type = &#8216;Train&#8217; / &#8216;Test&#8217;/ &#8216;Valid&#8217;</span></p>
<p style="text-align: left;"><span style="font-weight: 400;">Numerics = list of all numeric fields</span></p>
<p style="text-align: left;"><span style="font-weight: 400;">Categories = list of all categorical fields</span></p>
<p style="text-align: left;"><span style="font-weight: 400;">Features = list of all features except the Label and Id </span></p>
<pre>def convert_to_ffm(df,type,numerics,categories,features):
    currentcode = len(numerics)
    catdict = {}
    catcodes = {}
    
    # Flagging categorical and numerical fields
    for x in numerics:
         catdict[x] = 0
    for x in categories:
         catdict[x] = 1
    
    nrows = df.shape[0]
    ncolumns = len(features)
    with open(str(type) + "_ffm.txt", "w") as text_file:
    
    # Looping over rows to convert each row to libffm format
        for n, r in enumerate(range(nrows)):
             datastring = ""
             datarow = df.iloc[r].to_dict()
             datastring += str(int(datarow['Loan_Status'])) # Set Target Variable here
             
            # For numerical fields, we are creating a dummy field here
             for i, x in enumerate(catdict.keys()):
                 if(catdict[x]==0):
                     datastring = datastring + " "+str(i)+":"+ str(i)+":"+ str(datarow[x])
                 else:
            
            # For a new field appearing in a training example
                     if(x not in catcodes):
                         catcodes[x] = {}
                         currentcode +=1
                         catcodes[x][datarow[x]] = currentcode #encoding the feature
             
            # For already encoded fields
                     elif(datarow[x] not in catcodes[x]):
                         currentcode +=1
                         catcodes[x][datarow[x]] = currentcode #encoding the feature
                     
                     code = catcodes[x][datarow[x]]
                     datastring = datastring + " "+str(i)+":"+ str(int(code))+":1"

             datastring += '\n'
             text_file.write(datastring)</pre>
<p class="c2"><span class="c1">xLearn can handle csv as well as libsvm format for implementation of FMs while we necessarily need to convert it to libffm format for using FFM.<br />
</span></p>
<p class="c2"><span class="c1">Once we have the dataset in libffm format, we could train the model using the xLearn library.</span></p>
<p class="c2"><span class="c1">xLearn can automatically performs early stopping using the validation/test logloss and we can also declare another metric and monitor on the validation set for each iteration of the stochastic gradient descent.</span></p>
<p class="c2"><span class="c1">The following python script could be used for training and tuning hyperparameters of FFM model using xlearn on a dataset in ffm format. More options and complete documentation is given <a href="https://xlearn-doc.readthedocs.io/en/latest/cli_api/index.html">here</a>.</span></p>
<pre>ffm_model = xl.create_ffm()

ffm_model.setTrain("train_ffm.txt")

param = {'task':'binary', 
         'lr':0.2,
         'lambda':0.002, 
         'metric':'acc'}

# Start to train
# The trained model will be stored in model.out
ffm_model.fit(param, './model.out')

</pre>
<p style="text-align: left;"><span style="font-weight: 400;">The library also allows us to use cross-validation using the cv() function:</span></p>
<pre style="text-align: left;"><span style="font-weight: 400;">ffm_model.cv(param)</span></pre>
<p style="text-align: left;"><span style="font-weight: 400;">Predictions can be done on the test set with the following code snippet:</span></p>
<pre style="text-align: left;"><span style="font-weight: 400;"># Prediction task
ffm_model.setTest("test_ffm.txt") # Test data
ffm_model.setSigmoid() # Convert output to 0-1

# Start to predict
# The output result will be stored in output.txt
ffm_model.predict("./model.out", "./output.txt")
</span></pre>
