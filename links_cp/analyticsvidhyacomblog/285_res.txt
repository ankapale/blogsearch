<h1 itemprop="name" class="entry-title">How to use &#8220;VLOOKUP()&#8221; like functionality in QlikView?</h1>
<div class="meta-info">
<span class="entry-author">
<a href="https://www.analyticsvidhya.com/blog/author/sunil-ray/"><span>Sunil Ray</span></a>,
</span>
<time class="entry-date" content="2014-07-13" datetime="2014-07-13T20:22:54+00:00">
July 13, 2014 </time>
<div class="count-data right">
</div>
<span class="save-for-later"></span>
</div>
</header>
<div class="text-content">
<div id="ban3_av"></div><p style="text-align: justify;">Whenever I interact with a Qlikview user, who has migrated from Excel recently &#8211; one of the most common queries which comes through is:</p>
<h4 style="text-align: center;"><strong>&#8220;How do I apply VLOOKUP() in Qlikview?&#8221;</strong></h4>
<p style="text-align: justify;">For the starters, VLOOKUP is the excel way of joining 2 datasets through a common key. It is somewhat similar to joins in SQL.</p>
<p>&nbsp;</p>
<div id='stb-container-6283' class='stb-container-css stb-section-container stb-no-caption stb-image-none stb-ltr stb-corners stb-side-none'><aside class='stb-icon'><img src=''></aside><div id='stb-box-6283' class='stb-section_box stb-box'> <span style="color: #ffffff;"><strong>Why do we need an external VLOOKUP() like function in Qlikview?</strong></span></div></div>
<p style="text-align: justify;">Whenever you import an external dataset in Qlikview, it searches for common columns and performs joins on its own. Hence, people somewhat familiar with Qlikview might ask, why do we need a VLOOKUP() like function in the first place?</p>
<p style="text-align: justify;">The answer is, to build efficient data models. Let me explain with a real life situation, I had faced, in my initial days with Qlikview. I had been told by a few experienced people that, it is always advisable to minimize the number of tables in your data model. In my first assignment as a Qlikview developer, I used multiple joins and associations. As a result, my data model became too complex and difficult to understand. I was not able to identify and fix bugs easily (a similar data model example shown in the figure below for reference).</p>
<p style="text-align: justify;">As I spent more time with Qlikview, I came across the use of Mapping table with Applymap() function and I found them to be similar to VLOOKUP(), where we retrieve information from a mapping dataset to a primary dataset. This helps us simplify our data models quite significantly.</p>
<p>&nbsp;</p>
<p><a rel="lightbox" href=https://www.analyticsvidhya.com/blog/wp-content/uploads/2014/07/Fourth.png><img class="alignnone size-full wp-image-4857" src="https://www.analyticsvidhya.com/blog/wp-content/uploads/2014/07/Fourth.png" alt="Fourth" width="622" height="299" srcset="https://www.analyticsvidhya.com/wp-content/uploads/2014/07/Fourth.png 622w, https://www.analyticsvidhya.com/wp-content/uploads/2014/07/Fourth-300x144.png 300w" sizes="(max-width: 622px) 100vw, 622px" /></a></p>
<p>&nbsp;</p>
<p style="text-align: justify;">In Qlikview, joins can be performed using Association, Join and Applymap. Applymap works in a similar fashion as VLOOKUP with a few additional features. For example, if VLOOKUP does not find a match in mapping table, it returns &#8220;#NA&#8221;. Applymap, on the other hand, can return string / number / fields of similar table, in case there is no match. We can also write nested Applymap, to refer other mapping table, in case the value is not found.</p>
<p>&nbsp;</p>
<div id='stb-container-5562' class='stb-container-css stb-section-container stb-no-caption stb-image-none stb-ltr stb-corners stb-side-none'><aside class='stb-icon'><img src=''></aside><div id='stb-box-5562' class='stb-section_box stb-box'> <span style="color: #ffffff;"><strong>How can we define Mapping Table?</strong></span></div></div>
<p style="text-align: justify;">Mapping table must have two columns &#8211; first for the lookup key and second for the exchange value. Mapping tables are stored separately in memory and used only during script execution. After script execution, they are dropped automatically.</p>
<p style="color: #555555; text-align: justify;">The syntax to load a mapping table is similar to LOAD statement with an additional prefix (MAPPING) before LOAD. The source of mapping table can be a database, spreadsheet, resident table, inline table or a QVD file:</p>
<p style="color: #555555; text-align: justify;"><div id='stb-container-9045' class='stb-container-css stb-grey-container stb-no-caption stb-image-none stb-ltr stb-corners stb-side-none'><aside class='stb-icon'><img src='https://www.analyticsvidhya.com/blog/wp-content/plugins/wp-special-textboxes/images/gears-b.png'></aside><div id='stb-box-9045' class='stb-grey_box stb-box'>
<pre style="color: #555555;">Designation_desc:
 Mapping LOAD Designation_Code,
 Designation_Desc
 FROM
 Data_set.xlsx
 (ooxml, embedded labels, table is Designation);</pre>
<p style="text-align: justify;"></div></div>
<p style="text-align: justify;"><div id='stb-container-2406' class='stb-container-css stb-section-container stb-no-caption stb-image-none stb-ltr stb-corners stb-side-none'><aside class='stb-icon'><img src=''></aside><div id='stb-box-2406' class='stb-section_box stb-box'><strong><span style="color: #ffffff;">Applymap() function and Syntax:</span> </strong></div></div>
<p style="text-align: justify;"><span style="color: #3d3d3d;">The syntax is very similar to a lookup function &#8211; it takes one value as input, checks if this value exists in a mapping table and returns the corresponding value from the second column of the mapping table. The Applymap function is always used after defining a Mapping Table:</span></p>
<p><strong>Syntax:  </strong></p>
<p style="text-align: center;"><strong><span class="Bold">applymap(</span><span class="Italic">&#8216;mapname&#8217;,  lookupkey [ , defaultexpr ] </span><span class="Bold">)</span></strong></p>
<p>where:</p>
<p style="text-align: justify;"><strong><span class="Italic">mapname</span></strong> is the name of mapping table, that has been already created through the <span class="Bold">mapping load.</span> Its name must be enclosed in single quotes.</p>
<p style="text-align: justify;"><strong>lookupkey</strong> is the expression, the result of which should be mapped.</p>
<p style="text-align: justify;"><strong>defaultexpr</strong> is an optional expression, which will be used as default mapping value, if the mapping table does not contain a matching value for lookupkey. If no default value is given, the value of lookup will be returned as is.</p>
<p><a rel="lightbox" href=https://www.analyticsvidhya.com/blog/wp-content/uploads/2014/07/Applymap2.png><img class="alignnone wp-image-4858" src="https://www.analyticsvidhya.com/blog/wp-content/uploads/2014/07/Applymap2.png" alt="Applymap2" width="600" height="380" srcset="https://www.analyticsvidhya.com/wp-content/uploads/2014/07/Applymap2.png 1013w, https://www.analyticsvidhya.com/wp-content/uploads/2014/07/Applymap2-300x189.png 300w" sizes="(max-width: 600px) 100vw, 600px" /></a></p>
<div id='stb-container-9264' class='stb-container-css stb-section-container stb-no-caption stb-image-none stb-ltr stb-corners stb-side-none'><aside class='stb-icon'><img src=''></aside><div id='stb-box-9264' class='stb-section_box stb-box'><strong><span style="color: #ffffff;">How can we use Applymap?</span><strong> </strong></strong></div></div>
<p style="text-align: justify;">Let&#8217;s look at a scenario, where we have employee data set with their employee code, designation code, salary and Marital status. Further, you need employee designation description and Bonus details, which are available in a different table. You want to calculate total payout of an employee and report it along with his designation (Total Payout=Salary + Bonus). In SQL, you would join the required tables to get additional information. In Excel, we can use the Vlookup() function to do this. Here, we will use the Applymap function to map BONUS and DESIGNATION DESCRIPTION from respective mapping table.</p>
<p style="color: #3d3d3d;"><a rel="lightbox" href=https://www.analyticsvidhya.com/blog/wp-content/uploads/2014/07/First1.png><img class="alignnone size-full wp-image-4833" src="https://www.analyticsvidhya.com/blog/wp-content/uploads/2014/07/First1.png" alt="First" width="756" height="248" srcset="https://www.analyticsvidhya.com/wp-content/uploads/2014/07/First1.png 756w, https://www.analyticsvidhya.com/wp-content/uploads/2014/07/First1-300x98.png 300w" sizes="(max-width: 756px) 100vw, 756px" /></a></p>
<p style="color: #3d3d3d; text-align: justify;">Please note that this problem can also be solved through Join or Association. We will solve this using Applymap in this article.</p>
<p style="color: #3d3d3d; text-align: justify;"><strong>Step &#8211; 1:</strong></p>
<p style="color: #3d3d3d; text-align: justify;">In our QlikView script, we will first define the mapping tables:</p>
<ul style="text-align: justify;">
<li>Bonus &#8211; maps EmpCode to Bonus</li>
<li>Designation_desc &#8211; maps DesignationCode to designation description</li>
</ul>
<p style="color: #3d3d3d; text-align: justify;"><div id='stb-container-7224' class='stb-container-css stb-grey-container stb-no-caption stb-image-none stb-ltr stb-corners stb-side-none'><aside class='stb-icon'><img src='https://www.analyticsvidhya.com/blog/wp-content/plugins/wp-special-textboxes/images/gears-b.png'></aside><div id='stb-box-7224' class='stb-grey_box stb-box'>
<pre>Bonus:
 Mapping LOAD EmpCode,
 Bonus
 FROM Data_set.xlsx (ooxml, embedded labels, table is Bonus);</pre>
<pre>Designation_desc:
 Mapping LOAD DesignationCode,
 Designation_Desc
 FROM Data_set.xlsx (ooxml, embedded labels, table is Designation);</pre>
<p style="text-align: justify;"><span style="color: #555555;"></div></div></span></p>
<p style="text-align: justify;"><strong>Step &#8211; 2:</strong></p>
<p style="text-align: justify;">The second step is to use this information, when reading the Employee table:</p>
<p style="text-align: justify;"><span style="color: #555555;"><div id='stb-container-5538' class='stb-container-css stb-grey-container stb-no-caption stb-image-none stb-ltr stb-corners stb-side-none'><aside class='stb-icon'><img src='https://www.analyticsvidhya.com/blog/wp-content/plugins/wp-special-textboxes/images/gears-b.png'></aside><div id='stb-box-5538' class='stb-grey_box stb-box'></span></p>
<pre>Employee:
 LOAD EmpCode,
 Gender,
 MaritalStatus,
 DesignationCode,
 Salary,
 ApplyMap('Bonus',EmpCode,0) as Bonus,
 ApplyMap('Designation_desc',DesignationCode,'Unknown') as designation_Desc
 FROM Data_set.xlsx (ooxml, embedded labels, table is Main);</pre>
<p style="color: #3d3d3d; text-align: justify;"><span style="color: #555555;"></div></div></span></p>
<p style="color: #3d3d3d; text-align: justify;">Now, we have joined the fields Bonus and Designation description with the Employee dataset, without using a join or association.</p>
<p style="color: #3d3d3d; text-align: justify;"><div id='stb-container-9425' class='stb-container-css stb-section-container stb-no-caption stb-image-none stb-ltr stb-corners stb-side-none'><aside class='stb-icon'><img src=''></aside><div id='stb-box-9425' class='stb-section_box stb-box'><span style="color: #ffffff;"><strong>Benefits of using  ApplyMap:</strong></span></div></div>
<ul>
<li style="text-align: justify;">Multiple tables in your data model is not a good thing. So, you don’t want a situation where you have many lookups joined to your master table. This can be resolved by creating a mapping table using ApplyMap<span style="color: #747474;">. </span></li>
<li style="text-align: justify;">The mapping table only exists during load, we can call it as temporary table<span style="color: #747474;">.</span></li>
<li>ApplyMap has great features as its third (optional) parameter. The third parameter allows you to return something more useful, when lookup value not found in mapping table. Third parameter can be string, Number, field of base dataset, Mixture of string and field and more important reference to another mapping table using nested Applymap function<span style="color: #747474;">.</span>
<ul>
<li style="text-align: justify;"><strong>ApplyMap(&#8216;Bonus&#8217;,EmpCode,0)</strong>: Maps Bonus amount from Mapping table (Bonus). If Empcode does not exist in mapping table, then returns 0 (Zero).</li>
<li style="text-align: justify;"><strong>ApplyMap(&#8216;Designation_desc&#8217;,DesignationCode,&#8217;Unknown&#8217;): </strong>Map Designation description from Mapping table (Designation_desc). If DesignatonCode does not exist in mapping table, then returns &#8220;Unknown&#8221;.</li>
<li style="text-align: justify;"><strong>ApplyMap(&#8216;Bonus&#8217;,EmpCode,Salary*0.05)</strong>: Map Bonus amount from Mapping table (Bonus). If Empcode does not exist in mapping table, then it returns 5% of Salary (Field exists in Employee data set).</li>
<li style="text-align: justify;"><strong>ApplyMap(&#8216;Designation_desc&#8217;,DesignationCode, ApplyMap(&#8216;Bonus&#8217;,EmpCode,Salary*0.05)) : </strong>Map Designation description from Mapping table (Designation_desc). If DesignatonCode does not exist in mapping table, then it returns Bonus from Mapping Table (Bonus). Here we have used nested Applymap.</li>
</ul>
</li>
</ul>
<p style="color: #3d3d3d;"><a rel="lightbox" href=https://www.analyticsvidhya.com/blog/wp-content/uploads/2014/07/Second.png><img class="alignnone size-full wp-image-4841" src="https://www.analyticsvidhya.com/blog/wp-content/uploads/2014/07/Second.png" alt="Second" width="628" height="251" srcset="https://www.analyticsvidhya.com/wp-content/uploads/2014/07/Second.png 628w, https://www.analyticsvidhya.com/wp-content/uploads/2014/07/Second-300x119.png 300w" sizes="(max-width: 628px) 100vw, 628px" /></a></p>
<p style="color: #3d3d3d;"><a rel="lightbox" href=https://www.analyticsvidhya.com/blog/wp-content/uploads/2014/07/Third.png><img class="alignnone size-full wp-image-4842" src="https://www.analyticsvidhya.com/blog/wp-content/uploads/2014/07/Third.png" alt="Third" width="624" height="293" srcset="https://www.analyticsvidhya.com/wp-content/uploads/2014/07/Third.png 624w, https://www.analyticsvidhya.com/wp-content/uploads/2014/07/Third-300x140.png 300w" sizes="(max-width: 624px) 100vw, 624px" /></a></p>
<ul>
<li style="text-align: justify;">When mapping table has <span style="color: #747474;">duplicate rows, JOIN will lead to double counting where as  ApplyMap deals with the first instance of the key in the mapping table. It automatically ignores the subsequent duplicate values. Again, this is similar to how VLOOKUP() works in Excel.</span></li>
</ul>
<p style="color: #3d3d3d; text-align: justify;"><div id='stb-container-3373' class='stb-container-css stb-section-container stb-no-caption stb-image-none stb-ltr stb-corners stb-side-none'><aside class='stb-icon'><img src=''></aside><div id='stb-box-3373' class='stb-section_box stb-box'><span style="color: #ffffff;"> <strong>Concluding thoughts:</strong></span></div></div>
<p style="color: #3d3d3d; text-align: justify;">If we require multiple fields from reference table based on a key field, we can not do it through Applymap. We can have only two columns in a Mapping table. If we are required to map 4 or 5 fields, then I would prefer to create 4 or 5 mapping tables. However, if we need to join more than that, then I will look for another options like Join.</p>
<p style="color: #3d3d3d; text-align: justify;">This is an example where we can improve our data models and enhance the performance of QV dashboards. Do you have any experience / case studies illustrating the same in Qlikview? Do you have any other tip to share to make dashboards better?</p>
