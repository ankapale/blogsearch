<div id="ban3_av"></div><h2>Introduction</h2>
<p style="text-align: justify;">GraphLab came as an unexpected breakthrough on my learning plan. After all, ‘ Good Things Happen When You Expect Them Least To Happen’. It all started with the end of <a href="http://datahack.analyticsvidhya.com/contest/black-friday-data-hack" target="_blank" rel="nofollow">Black Friday Data Hack</a>. Out of 1200 participants, we got our winners and their interesting <a href="https://www.analyticsvidhya.com/blog/2015/11/secrets-from-data-hackathon/" target="_blank" rel="nofollow">solutions</a>.</p>
<p style="text-align: justify;">I read and analyzed them. I realized that I had missed on an incredible machine learning tool. A quick exploration told me that this tool has immense potential to reduce our machine learning modeling pains. So, I decided to explore it further. I now have dedicated a few days to understand its science and logical methods of usage. To my surprise, it wasn’t difficult to understand.</p>
<p style="text-align: justify;">Were you trying to improve your Machine Learning model ? But failed mostly? Try this advanced machine learning tool. A month trial is free and 1 year subscription is available for FREE for academic use. Then, you can purchase subscription for following years.</p>
<p style="text-align: justify;">To get you started quickly, here is a beginners guide on GraphLab in Python. For ease of understanding, I’ve tried to explain these concepts in simplest possible manner.</p>
<p>&nbsp;</p>
<h2>Topics Covered</h2>
<ol>
<li style="text-align: justify;">How it all started ?</li>
<li style="text-align: justify;">What is GraphLab ?</li>
<li style="text-align: justify;">Benefits and Limitations of GraphLab</li>
<li style="text-align: justify;">How to install GraphLab ?</li>
<li style="text-align: justify;">Getting started with GraphLab</li>
</ol>
<p>&nbsp;</p>
<h2>How it all started ?</h2>
<p>GraphLab has an interesting story of its inception. Let me tell you in brief.</p>
<p style="text-align: justify;">GraphLab, known as Dato is founded by <a href="https://www.linkedin.com/in/carlos-guestrin-5352a869" target="_blank" rel="nofollow">Carlos Guestrin</a>. Carlos holds Ph.D in Computer Science from Stanford University.  It happened around 7 years back. Carlos was a professor at Carnegie Mellon University. Two of his students were working on large scale distributed machine learning algorithms. They ran their model on top of Hadoop and found it took quite long to compute. Situations didn’t even improve after using <a href="https://en.wikipedia.org/wiki/Message_Passing_Interface" target="_blank" rel="nofollow">MPI</a> (high performance computing library).</p>
<p style="text-align: justify;">So, they decided to build a system to write more papers quickly. With this, GraphLab came into existence.</p>
<p style="text-align: justify;"><strong>P.S</strong> &#8211; GraphLab Create is a commercial software by GraphLab. GraphLab Create is accessed in python using &#8220;graphlab&#8221; library. Hence, in this article, &#8216;GraphLab&#8217; connotes GraphLab Create. Don&#8217;t get confused.</p>
<p>&nbsp;</p>
<h2>What is GraphLab?</h2>
<p style="text-align: justify;">GraphLab is a new parallel framework for machine learning written in C++. It is an open source project and has been designed considering the scale, variety and complexity of real world data. It incorporates various high level algorithms such as Stochastic Gradient Descent (SGD), Gradient Descent &amp; Locking to deliver high performance experience. It helps data scientists and developers easily create and install applications at large scale.</p>
<p style="text-align: justify;">But, what makes it amazing?  It’s the presence of neat libraries for data transformation, manipulation and model visualization. In addition, it comprises of scalable machine learning toolkits which has everything (almost) required to improve machine learning models. The toolkit includes implementation for deep learning, factor machines, topic modeling, clustering, nearest neighbors and more.</p>
<p>Here is the complete architecture of GraphLab Create.</p>
<p><a rel="lightbox" href=https://www.analyticsvidhya.com/wp-content/uploads/2015/12/architechture.png><img class="size-full wp-image-21682 aligncenter" src="https://www.analyticsvidhya.com/wp-content/uploads/2015/12/architechture.png" alt="architechture" width="692" height="449" srcset="https://www.analyticsvidhya.com/wp-content/uploads/2015/12/architechture.png 692w, https://www.analyticsvidhya.com/wp-content/uploads/2015/12/architechture-300x195.png 300w" sizes="(max-width: 692px) 100vw, 692px" /></a></p>
<p>&nbsp;</p>
<h2>What are the Benefits of using GraphLab ?</h2>
<p style="text-align: justify;">There are multiple benefits of using GraphLab as described below:</p>
<ul style="text-align: justify;">
<li><strong>Handles Large Data: </strong>Data structure of GraphLab can handle large data sets which result into scalable machine learning. Let&#8217;s look at the data structure of Graph Lab:
<ul>
<ul>
<li><strong><a href="https://dato.com/learn/gallery/notebooks/introduction_to_sframes.html" target="_blank" rel="nofollow">SFrame</a>: </strong>It is an efficient disk-based tabular data structure which is not limited by RAM. It helps to scale analysis and data processing to handle large data set (Tera byte), even on your laptop. It has similar syntax like pandas or R data frames. Each column is an <strong>SArray</strong>, which is a series of elements stored on disk. This makes SFrames disk based. I have discussed the methods to work with &#8220;SFrames&#8221; in following sections.</li>
<li><strong><a href="https://dato.com/products/create/docs/generated/graphlab.SGraph.html" target="_blank" rel="nofollow">SGraph</a>:</strong> Graph helps us to understand networks by analyzing relationships between pair of items. Each item is represented by a <em>vertex</em> in the graph. Relationship between items is represented by <em>edges</em>. In GraphLab, to perform a graph-oriented data analysis, it uses <a href="https://dato.com/products/create/docs/generated/graphlab.SGraph.html" target="_blank">SGraph</a> object. It is a scalable graph data structure which store vertices and edges in SFrames. To know more about this, please refer this <a href="https://dato.com/products/create/docs/generated/graphlab.SGraph.html" target="_blank" rel="nofollow">link</a>. Below is graph representation of James Bond characters.<br />
<a rel="lightbox" href=https://www.analyticsvidhya.com/wp-content/uploads/2015/12/bond_basic.png><img class="aligncenter wp-image-21678 " src="https://www.analyticsvidhya.com/wp-content/uploads/2015/12/bond_basic.png" alt="bond_basic" width="590" height="293" srcset="https://www.analyticsvidhya.com/wp-content/uploads/2015/12/bond_basic.png 765w, https://www.analyticsvidhya.com/wp-content/uploads/2015/12/bond_basic-300x149.png 300w" sizes="(max-width: 590px) 100vw, 590px" /></a></li>
</ul>
</ul>
</li>
<li><strong>Integration with various data sources: </strong>GraphLab supports various data sources like S3, ODBC, JSON, CSV, HDFS and many more.</li>
<li><strong>Data exploration and visualization with GraphLab Canvas</strong>. GraphLab Canvas is a browser-based interactive GUI which allows you to explore tabular data, summary statistics and bi-variate plots. Using this feature, you spend less time coding for data exploration. This will help you to focus more on understanding the relationship and distribution of variables. I have discussed this part following sections.</li>
<li><strong>Feature Engineering: </strong>GraphLab has an inbuilt option to create new useful features to enhance model performance. It comprises of various options like transformation, binning, imputation, One hot encoding, tf-idf etc.</li>
<li><strong>Modeling:</strong> GraphLab has various toolkits to deliver easy and fast solution for ML problems. It allows you to perform various modeling exercise (regression, classification, clustering) in fewer lines of code. You can work on problems like recommendation system, churn prediction, sentiment analysis, image analysis and many more.</li>
<li><strong>Production automation:</strong> Data pipelines allow you to assemble reusable code task into jobs. Then, automatically run them on common execution environments (e.g. Amazon Web Services, Hadoop).</li>
<li><strong>GraphLab Create SDK:</strong> Advance users can extend the capabilities of GraphLab Create using GraphLab Creat SDK. You can define new machine learning models/ programs  and integrate them with the rest of the package. See the GitHub repository <a href="https://github.com/dato-code/GraphLab-Create-SDK." target="_blank" rel="nofollow">here</a>.</li>
<li><strong>License: </strong>It has limitation to use. You can go for 30 days free trial period or one year license for academic edition. To extend your subscription you;ll be charged (see subscription structure <a href="https://dato.com/buy/" target="_blank" rel="nofollow">here</a>).</li>
</ul>
<p>&nbsp;</p>
<h2>How to Install GraphLab?</h2>
<p style="text-align: justify;"><span data-reactid=".fo1ua8pwcg.0.0.0.0.0.0.0">You can also use GraphLab once you have availed its license. However, you can also get started with free trial</span><span data-reactid=".fo1ua8pwcg.0.0.0.0.0.0.2"> or the academic edition with 1 year subscription. So, prior to installation, your machine must fulfill the system requirement to run GraphLab.</span></p>
<p style="text-align: justify;"><strong>System Requirement for GraphLab:</strong></p>

<h2>Getting started with Graphlab</h2>

<p style="text-align: justify;">Here, I&#8217;ll demonstrate the use of GraphLab by solving  a data science challenge. I have the taken data set from <a href="http://datahack.analyticsvidhya.com/contest/black-friday-data-hack" target="_blank" rel="nofollow">Black Friday Data Hack</a>.</p>

 if age=='51-55':
 return '50+'

<li><strong>Feature Engineering: </strong>Feature engineering is an efficient method to improve model performance. Using this technique, we can create new variable(s) after transformation or manipulation of existing existing variable(s). In fact, GraphLab has automated this process. They have various transformation options for numerical, categorical, text and image features. Also, you&#8217;ll find direct options for feature binning, imputation, one hot encoding, Count threshold, TF-IDF, Hasher, Tokenzing and others. Let&#8217;s look at the imputation of categorical feature &#8220;<strong>Product_Category_2</strong>&#8221; based on &#8220;<strong>Age</strong>&#8221; and &#8220;<strong>Gender</strong>&#8221; of &#8220;Black Friday&#8221; data set.
<pre># Create the data
# Variables based on which we want to perform imputation and variable to impute
# You can look at the algorithms behind the imputation <a href="https://dato.com/learn/userguide/feature-engineering/categorical_imputer.html" target="_blank" rel="nofollow">here</a>.

sf_impute = sf_train['Age','Gender','Product_Category_2']</pre>
<pre>imputer = graphlab.feature_engineering.CategoricalImputer(feature='Product_Category_2')
# Fit and transform on the same data
transformed_sf = imputer.fit_transform(sf_impute)</pre>
<pre>#Retrieve the imputed values
transformed_sf

<a rel="lightbox" href=https://www.analyticsvidhya.com/wp-content/uploads/2015/12/Impute_Categorical.png><img class="aligncenter wp-image-21675 " src="https://www.analyticsvidhya.com/wp-content/uploads/2015/12/Impute_Categorical.png" alt="Impute_Categorical" width="573" height="271" srcset="https://www.analyticsvidhya.com/wp-content/uploads/2015/12/Impute_Categorical.png 749w, https://www.analyticsvidhya.com/wp-content/uploads/2015/12/Impute_Categorical-300x142.png 300w" sizes="(max-width: 573px) 100vw, 573px" /></a></pre>

<pre>sf_train['Predicted_Product_Category_2']=transformed_sf['predicted_feature_Product_Category_2']</pre>

<li><strong><strong>Modeling: </strong></strong>At this stage, we do predictions from the past data. GraphLab easily create models for common tasks, such as:<br />
A) Predicting Numeric Quantities<br />
B) Building Recommendation Systems<br />
C) Clustering Data and Documents<br />
D) Analyzing Graphs</li>

<p>In Black Friday challenge, we are required predict the numeric quantities &#8220;Purchase&#8221; i.e. we need a regression model to predict the &#8220;Purchase&#8221;.</p>
<p>In GraphLab, we have three type of regression models:<br />
A) Linear Regression<br />
B) Random Forest Regression<br />
C) Gradient Boosted Regression<br />
If you have any confusion in algorithm selection, GraphLab takes care of that. Don&#8217;t worry. It selects the right regression model automatically.</p>
<pre># Make a train-test split
train_data, validate_data = sf_train.random_split(0.8)</pre>
<pre># Automatically picks the right model based on your data.
model = graphlab.regression.create(train_data, target='Purchase', features = ['Gender','Age','Occupation','City_Category','Stay_In_Current_City_Years',
 'Marital_Status','Product_Category_1'])</pre>
<pre># Save predictions to an SArray
predictions = model.predict(validate_data)</pre>
<pre># Evaluate the model and save the results into a dictionary
results = model.evaluate(validate_data)
results</pre>
<p><strong><strong>Output:<br />
</strong></strong></p>
<pre>{'max_error': 13377.561969523947, 'rmse': 3007.1225949345117}</pre>
<pre>#Do prediction on test data set
final_predictions = model.predict(sf_test)</pre>
